# 统一文件管理系统产品功能设计文档

## 文档信息

- 文档名称：统一文件管理系统产品功能设计文档
- 文档用途：用于后续功能设计、技术架构、实现拆解与 AI 协同开发对齐
- 当前阶段：首版产品功能设计
- 首发平台：Windows 桌面端
- 目标版本：MVP / V1

## 1. 产品定位

本项目是一套以“文件”为核心的统一文件管理系统，目标是统一管理分散在本机磁盘、移动硬盘、局域网 SMB/NAS、115 网盘等多个存储端点上的文件资产。

与传统文件管理系统以“磁盘、挂载点、文件夹”为管理核心不同，本系统以“统一文件资产”为核心进行组织。对于用户而言，系统首先回答的是“我有哪些文件”，其次才是“这些文件存在哪些存储端点上、状态如何、还能做什么操作”。

本系统主要面向个人创作者、摄影师、自媒体从业者和小型工作室等轻量化专业场景，重点解决以下问题：

- 文件分散在多种介质和平台中，整体不可见。
- 用户经常不知道某个文件当前在哪、是否已备份、是否已同步。
- 传统文件夹式管理方式难以承载海量照片、视频、音频等媒体资产管理需求。
- 从移动硬盘/U 盘导入、跨端同步、两地三介质管理过程复杂，容易漏拷、错拷、重复拷贝。

产品长期目标不是管理“介质”本身，而是让用户更方便地实现跨端统一文件管理，并逐步把原本在系统文件夹中完成的大部分文件操作迁移到本系统中完成。

## 2. 产品目标

### 2.1 总体目标

打造一套围绕“统一文件资产表”运行的桌面文件管理系统，让用户在一个系统内完成文件查看、检索、组织、导入、同步、复制、删除和状态确认等核心操作。

### 2.2 首版目标

- 支持用户创建多个资产库，并分别配置其存储端点。
- 支持接入本机磁盘、移动硬盘/U 盘、局域网 SMB/NAS、115 网盘。
- 支持统一索引和管理全部文件资产，不再以某个单一端点为主视角。
- 支持照片、视频、音频的精细化元数据管理。
- 支持重命名、移动、复制、删除、批量操作、目录视图等基础文件管理能力。
- 支持从移动硬盘/U 盘向多个目标端点执行拷贝入库。
- 支持跨端同步，且必须支持断点续传。
- 支持文件校验，降低漏拷、错拷、损坏不可见等风险。
- 支持冲突与异常集中提醒，由用户统一处理。

### 2.3 长期目标

- 扩展到 macOS 与 Web 管理后台。
- 扩展更多国内主流网盘及 S3 对象存储。
- 支持更丰富的自动归档规则和存储策略。
- 逐步成为用户日常主工作流中的统一文件入口，而非辅助工具。

## 3. 目标用户

### 3.1 核心用户

- 摄影师：管理大量 RAW、JPEG、视频片段、成片与交付文件。
- 自媒体创作者：管理拍摄素材、录音、视频工程文件、导出文件与封面素材。
- 个人创作者：管理长期积累的多类型创作资产。
- 小型工作室中的单用户或轻协作角色：需要高效管理素材与交付文件，但不需要企业级协同系统。

### 3.2 用户特征

- 文件数量多、体量大、格式复杂。
- 存储端点分散，常见于本地磁盘、移动盘、NAS、网盘并存。
- 对可靠性敏感，不能接受漏拷、误删、损坏不可见。
- 需要比通用资源管理器更强的媒体信息管理能力。
- 希望少进原始文件夹，多在统一系统中完成文件管理。

## 4. 产品设计原则

- 文件优先：以文件资产为中心建模，而不是以存储端点为中心建模。
- 单一事实源：一个资产库内针对单个文件只维护一条统一文件资产记录，多个存储端上的存在状态归属于同一条资产记录。
- 明确不模糊：同文件判定不引入“疑似同文件”概念，系统只输出明确结果。
- 媒体优先优化：所有文件都能管理，但照片、视频、音频的管理颗粒度必须更细。
- 结果可验证：所有导入、复制、同步类任务都应具备可校验、可恢复、可追踪的任务语义。
- 操作可替代：首版就要尽可能替代用户原先在 Windows 文件夹中的基础文件管理行为。
- 异常集中治理：冲突、失败、临时文件、异常状态不应散落在各流程中，应统一收敛到可处理入口。
- 能力可扩展：首版可以先以手动分发为主，但模型和任务系统必须为后续自动化规则预留扩展位。

### 5.1 首版范围

- 资产库管理
- 存储端管理与挂载选择
- 统一文件资产索引
- 统一文件列表与目录视图
- 搜索、筛选、排序
- 媒体元数据解析与展示
- 文件详情与多端状态查看
- 去重识别
- 基础文件管理能力
- 拷贝入库
- 跨端复制与同步
- 断点续传
- 校验与任务结果确认
- 删除与逻辑清理语义
- 冲突与异常处理中心

### 5.2 明确不做

- 企业级 OA、审批流
- 团队知识库

- 在线协作文档
- 多组织、多角色、多权限复杂协同

## 6. 核心概念模型

### 6.1 资产库

资产库是用户管理文件资产的逻辑边界。用户首次使用时先创建资产库，再为该资产库配置需要纳管的存储端点。

一个用户可以拥有多个资产库，例如：

- 商业摄影资产库
- 自媒体视频资产库
- 家庭照片资产库

### 6.2 存储端

存储端是系统中可接入、可读写、可执行同步与导入操作的目标位置。首版支持以下存储端类型：

- 本机磁盘目录
- 移动硬盘/U 盘目录
- 局域网 SMB/NAS 共享目录
- 115 网盘

系统应提供“存储端管理”能力，客户端当前按以下三类信息分开展示和维护：

- 挂载文件夹管理：展示某个资产库实际挂载到哪个本地目录、NAS 子目录或网盘目录
- NAS 管理：维护 NAS 主机配置
- 网盘管理：维护网盘接入配置

用户可预先配置 NAS / 网盘来源，再在资产库层选择实际挂载目录。移动硬盘 / U 盘接入提醒优先进入通知中心，由用户决定是否后续纳入挂载管理，而不是作为客户端中的独立类型常驻维护。

### 6.3 统一文件资产

统一文件资产是系统中的核心实体。一个资产库中的一个文件资产只存在一条主记录，用于表达：

- 文件的主身份信息
- 文件所属目录关系
- 文件类型与媒体属性
- 元数据
- 标签与组织信息
- 在不同存储端上的存在状态
- 最近任务状态

### 6.4 存储副本状态

同一统一文件资产可对应多个存储副本状态，用于表达其在不同存储端上的状态，例如：

- 已存在
- 缺失
- 待同步
- 同步中
- 校验失败

### 6.5 导入任务

导入任务指从移动硬盘/U 盘或其他来源端，将文件拷贝到资产库绑定的一个或多个目标存储端，并将结果纳入统一文件资产表的过程。

### 6.6 同步任务

同步任务指在两个或多个存储端之间对已有文件资产执行复制、补齐、恢复或状态对齐的过程。同步任务必须支持断点续传。

### 6.7 异常待处理项

异常待处理项用于统一承接各种需人工处理的问题，例如：

- 文件名冲突
- 路径冲突
- 源端和目标端均发生变更
- 存储空间不足
- 网络中断后未恢复
- 历史遗留临时文件（如 `.tmp` 等）

## 7. 关键业务规则

### 7.1 同文件判定规则

系统参考 FreeFileSync 的实现思路，但首版以工程可控、结果明确为原则，主要基于以下因素判定同文件：

- 在各存储端中的相对路径
- 文件名称
- 文件大小
- 文件类型
- 必要的时间属性与状态属性

系统不对用户暴露“疑似同文件”状态。若系统无法依据规则得出明确结论，应进入异常待处理项，由系统提示人工介入，而不是自动合并或输出不确定结论。

### 7.2 统一文件资产主键策略

统一文件资产的主身份应围绕“相对路径 + 名称 + 大小 + 类型 + 必要状态信息”的组合规则构建，服务于跨端识别与状态归并。实现层可采用内部技术主键，但业务识别规则必须与同文件判定规则保持一致。

### 7.3 目录关系规则

- 系统既要提供统一文件视图，也要保留目录视图。
- 目录视图用于承接用户从传统文件夹管理迁移过来的操作习惯。
- 目录变更应映射到统一文件资产的组织关系中，而不是绕开资产表直接只改某一端点。

### 7.4 删除规则

- 首版必须支持“删除资产”和“从指定端点删除”两类删除操作。
- 删除后不进入独立缓冲区；被删除副本在正式状态中表现为缺失。
- 只要仍有任一副本存在，资产继续保留在系统中。
- 当所有副本都不存在时，资产进入逻辑删除标记，并等待后台清理任务处理。

### 7.5 同步与复制规则

- 同步和复制都应具备任务化执行能力。
- 大文件和长任务必须支持断点续传。
- 任务中断后可恢复，恢复时应尽量复用已有传输结果。
- 任务完成后必须支持结果校验。

### 7.6 异常处理规则

以下问题应统一收敛并提醒用户处理：

- 文件名冲突
- 路径冲突
- 双端均发生变更
- 目标端空间不足
- 网络异常
- 鉴权失效
- 历史遗留临时文件

系统处理原则应接近用户熟悉的文件夹复制体验，例如在重名时支持跳过，但应提供更强的集中治理能力和可追溯信息。

## 8. 信息架构

建议首版桌面端采用如下一级模块：

- 资产库切换
- 文件中心
- 导入中心
- 任务中心
- 异常中心
- 存储节点
- 设置

其中当前客户端已经落地的主工作区为 `文件中心`：

- 文件总览与目录导航当前合并在同一页面中完成，不再拆成两个一级模块
- 导入中心仍为独立入口，但文件中心页面内已提供“上传文件 / 上传文件夹”快捷入口
- 同步能力当前主要由文件中心内的端点状态按钮、批量工具条和更多操作菜单发起
- 设置页当前已新增独立“标签管理”子页面，位于“文件总览”之后、“校验恢复”之前，用于全局标签库管理；文件中心仍保留标签编辑弹层作为资产快速整理入口

## 9. 功能设计

### 9.1 首次使用与资产库创建

#### 功能目标

帮助用户完成首个资产库建立，并配置后续统一管理所需的存储端点。

#### 功能要求

- 首次进入系统时，若无资产库，优先引导创建资产库。
- 创建资产库时填写基础信息，如名称、说明、默认目录策略等。
- 在资产库创建过程中可直接选择已配置存储端，或新建存储端。
- 一个资产库可绑定多个存储端。
- 资产库支持后续继续添加和移除存储端。

#### 价值

明确“一个资产库就是一个独立的文件管理空间”，为后续统一资产表、任务系统和规则系统提供边界。

### 9.2 存储端管理

#### 功能目标

提供统一的存储端接入与管理能力，作为资产库配置与任务执行的底座。

#### 功能要求

- 支持新增、编辑、删除挂载文件夹、NAS 配置和网盘配置。
- 支持为挂载文件夹选择所属资产库。
- 支持本地目录、NAS 子目录、网盘目录三类挂载方式。
- 支持配置 SMB/NAS 地址、账号密码与网盘鉴权信息。
- 支持检测连通性、读写权限、剩余空间、最近状态。
- 支持将挂载文件夹标记为只读或可写。
- 支持复用已配置的 NAS / 网盘来源给多个挂载目录。
- 后续扩展网盘和对象存储时，接入方式需保持一致抽象。

#### 客户端页面实现约束

- 客户端存储端管理页面拆分为 `挂载文件夹管理`、`NAS 管理`、`网盘管理` 3 个子页。
- `挂载文件夹管理` 为主工作区，展示与资产库绑定的具体挂载目录。
- `NAS 管理` 与 `网盘管理` 用于维护来源配置，不直接替代挂载目录管理。
- 多选批量操作当前只保留批量扫描与批量连接测试，不提供批量设置心跳。

#### 设计要求

- 接入网盘时优先复用成熟开源工具或能力，不将上传下载细节完全自研为首选路径。
- 存储端能力层需要屏蔽不同协议差异，为上层任务系统提供统一访问方式。

### 9.3 文件中心

#### 功能目标

提供资产库下全部文件的统一查看与目录式导航入口，作为当前客户端的主工作台。

#### 功能要求

- 支持资产库切换后的目录列表加载。
- 支持面包屑导航，以及目录前进 / 后退。
- 支持关键字搜索，当前搜索范围为：名称、路径、来源端。
- 支持文件类型筛选：`全部 / 文件夹 / 图片 / 视频 / 音频 / 文档`。
- 支持存储状态筛选：`全部 / 完全同步 / 部分同步 / 未同步`。
- 支持“部分同步”按端点组合筛选，当前组合关系为 `AND`。
- 支持排序字段：`修改时间 / 名称 / 大小 / 星级`，并支持正序 / 逆序切换。
- 支持分页浏览，当前每页数量为 `10 / 20 / 50 / 100`。
- 支持批量选择与批量操作。
- 支持直接查看每个文件或目录在各受管端点上的状态。
- 支持在当前目录新建目录、上传文件、上传文件夹、刷新索引。

#### 设计重点

文件中心不是传统资源管理器式的裸目录列表，而是“统一资产 + 目录导航”的混合工作台。当前实现仍以表格列表为主，不提供独立缩略图视图，也未提供独立目录树侧栏。

### 9.4 目录导航与目录操作

#### 功能目标

在统一资产模型之上保留接近传统文件夹的使用习惯，降低迁移成本。

#### 功能要求

- 当前不提供独立目录树，目录导航集成在文件中心页头的面包屑与前进 / 后退中。
- 支持双击目录进入下一级目录。
- 支持在当前目录新建目录。
- 目录与文件共用同一张列表，目录项会展示派生后的端点状态与子项目数量。
- 目录级同步与删除副本会递归作用到所有可操作的后代文件。
- 当前客户端尚未实现目录重命名、目录移动与目录复制。

#### 设计重点

目录导航仍然落在统一文件资产体系之上，而不是独立于资产体系再维护一套目录页。

### 9.5 文件详情

#### 功能目标

在单文件粒度上展示当前可用的资产信息、元数据与人工标记。

#### 功能要求

- 展示基础信息：名称、路径、大小、类型、创建时间、修改时间。
- 展示来源、最近任务、元数据行、备注、标签徽标、风险标签。
- 支持在详情抽屉中编辑星级与色标。
- 标签编辑当前通过独立弹层完成，不放在详情抽屉内直接编辑。
- 跨端同步与删除副本当前主要在列表页完成，而不是在详情抽屉内集中完成。
- 当前客户端尚未实现详情内的重命名、移动、复制、打开所在目录。

### 9.6 媒体元数据管理

#### 功能目标

针对照片、视频、音频提供细粒度元数据管理能力，达到类似 Adobe Bridge 的管理深度。

#### 首版支持方向

- 照片：EXIF、拍摄时间、设备信息、镜头信息、分辨率、色彩空间、方向等。
- 视频：拍摄时间、分辨率、时长、编码参数、帧率、音视频流信息等。
- 音频：时长、编码格式、采样率、声道、比特率等。
- 通用：标签、文件类型、大小、来源端、导入时间、最近任务状态。

#### 功能要求

- 支持元数据解析、展示、筛选、排序。
- 支持在设置页通过独立“标签管理”子页面管理标签，文件中心仍保留原有标签编辑弹层作为使用入口。
- 当前客户端中的“标签管理”子页面已实现顶部统计徽标、分组侧栏、标签列表、详情面板、新增分组 / 新增标签弹层，以及合并 / 删除标签二次确认流程。
- 支持围绕媒体属性进行检索和组织。

### 9.7 搜索与筛选

#### 功能目标

让用户以“资产条件”而非“目录位置”快速找到目标文件。

#### 功能要求

- 支持关键字搜索，当前搜索范围为名称、路径、来源端。
- 支持文件类型与存储状态的组合筛选。
- 支持“部分同步”按端点名称进一步细分。
- 支持按修改时间、名称、大小、星级排序，并支持方向切换。
- 当前未实现常用筛选条件保存。
- 当前未实现按扩展名、拍摄时间、设备、分辨率、时长等高级筛选入口。

### 9.8 基础文件管理

#### 功能目标

在系统内承接用户原本在 Windows 文件夹中的高频操作。

#### 首版必须包含

- 新建目录
- 上传文件
- 上传文件夹
- 删除
- 批量操作
- 目录导航

#### 设计要求

- 操作对象既可以是单文件，也可以是批量文件。
- 操作结果需要同步反映到统一文件资产表和存储副本状态。
- 对于跨端操作，优先走任务化流程，以确保可恢复与可校验。

#### 当前客户端已实现范围

- 单项操作：查看详情、标签编辑、同步到指定端点、从指定端点删除副本、删除资产。
- 批量操作：批量标记、批量同步到指定端点、批量删除指定端点副本、批量删除资产、清空选择。
- 删除副本时，若该资产已无剩余受管副本，资产会自动从文件中心移除。
- 当前客户端尚未实现重命名、移动、复制。

### 9.9 去重识别

#### 功能目标

帮助用户识别和管理同一资产在多个端点的存在情况，以及不必要的重复存储。

#### 功能要求

- 自动识别符合规则的同文件并归并到统一文件资产。
- 明确展示某资产在哪些端点存在。
- 支持筛选仅存在于单端、存在于多端、缺少某关键端副本的文件。
- 不输出“疑似重复”作为正式状态。

### 9.10 导入中心

#### 功能目标

承接从移动硬盘/U 盘等来源端进入资产库的高频入库需求。

#### 首版流程

- 用户选择来源端。
- 用户在导入界面中，将不同文件拖入不同目标端。
- 用户启动导入任务。
- 系统执行拷贝、断点续传、校验和状态更新。

#### 功能要求

- 支持同时导入到多个目标端。
- 支持任务进度、速度、剩余时间、失败重试、断点续传。
- 支持导入后校验。
- 支持导入结果汇总。
- 对漏拷、失败、冲突、异常进行统一提示。

#### 扩展要求

首版以手动分发为主，但任务和规则模型需为后续自动归档规则预留能力。

### 9.11 同步中心

#### 功能目标

承接任一存储端到另一存储端的跨端同步与补齐。

#### 功能要求

- 当前客户端支持基于文件、目录、当前筛选结果和批量选择发起同步。
- 当前同步入口包括：端点状态按钮、行内“更多操作 -> 同步”、批量工具条“同步”。
- 当前 UI 不要求用户手工选择源端；源端选择由底层规则决定，页面只选择目标端。
- 支持断点续传。
- 支持任务中断恢复。
- 支持同步后校验。
- 支持按资产状态补齐副本，例如“补齐所有未同步到 NAS 的文件”。

### 9.12 任务中心

#### 功能目标

统一承接所有长任务的执行、跟踪和恢复。

#### 覆盖任务类型

- 扫描任务
- 元数据解析任务
- 导入任务
- 复制任务
- 同步任务
- 校验任务
- 删除与清理任务

#### 功能要求

- 支持查看任务状态、进度、速度、耗时、错误原因。
- 支持暂停、继续、取消、重试。
- 支持任务恢复。
- 传输任务支持按资产库、业务类型、状态筛选；其它任务当前支持按任务类型、状态、排序与搜索筛选，不提供独立资产库筛选。

#### 当前客户端原型已对齐

- 当前任务中心已将任务模型与文件数据关联：一级任务使用 `fileNodeIds`，二级任务使用 `fileNodeId`；名称、路径、大小优先从 `fileNodes` 实时解析，导入或未入库场景再兜底旧字段。
- 文件夹型任务展开时，二级项按递归文件展开，不再展示文件夹节点本身。
- 任务状态筛选已新增“活跃中”默认项。
- 任务行动作已按互斥规则渲染：暂停 / 继续 / 重试三者互斥，已完成显示“完成”占位按钮。
- 已删除关联文件的任务子项会进入失效展示态，表现为文件已删除、删除线样式且整行不可点击。
- 客户端启动时会用最新任务 mock 覆盖任务相关持久化状态；删除文件前也会先检查关联运行中传输任务，存在冲突时会提示，并可跳转任务或取消任务后再删除。
- 当前任务中心保留“传输任务 / 其它任务”顶层切换；“其它任务”当前只纳入扫描、元数据解析、校验、删除清理四类非传输后台任务，对账扫描、索引任务、修复 / 恢复任务暂不进入该子页面。
- “其它任务”主列表当前不区分一二级任务，文件、目录、资产等对象只在详情抽屉和异常联动中承接。
- “其它任务”当前支持任务类型筛选、状态筛选、排序、搜索、批量暂停 / 继续 / 取消 / 重试 / 优先级调整；不提供独立资产库筛选。
- 删除清理任务当前会明确展示“等待清理 / 清理中”阶段，并支持在 `运行中` 与 `等待清理` 状态下执行暂停，暂停后显示“继续”。
- 存储节点页发起的扫描任务当前会统一进入“任务中心 > 其它任务”跟踪；文件中心提交的删除请求会在同页以“删除清理任务”继续跟踪。

### 9.13 异常中心

#### 功能目标

提供统一的问题收敛入口，让用户集中处理各类风险和冲突。

#### 应纳入的异常类型

- 文件名冲突
- 路径冲突
- 双端变更冲突
- 临时文件待处理
- 校验失败
- 网络异常
- 鉴权失效
- 空间不足

#### 功能要求

- 每类异常可查看数量、影响范围、建议动作。
- 支持跳过、保留、删除、重命名、重试等处理方式。
- 支持后续补处理，而不要求当场全部解决。

## 10. 页面级设计建议

### 10.1 资产库首页

建议展示：

- 资产总数
- 总容量
- 各类型文件分布
- 各存储端状态摘要
- 最近任务
- 待处理异常

### 10.2 文件中心工作台

当前客户端已实现为以下区域：

- 顶部页头：资产库切换、导入中心入口、通知入口
- 页面头部：目录前进 / 后退、面包屑、搜索框
- 工具条：文件类型筛选、状态筛选、排序、上传、新建目录、刷新索引
- 选择工具条：批量标记、批量同步、批量删除副本、批量删除资产、清空选择
- 主体区：文件 / 目录表格列表
- 弹层区：文件详情抽屉、标签编辑弹层、批量标记弹层、确认同步 / 删除弹窗、新建目录抽屉

### 10.3 导入工作台

建议重点突出：

- 来源端选择
- 目标端分发区
- 待导入文件区
- 任务预检结果
- 启动导入按钮

### 10.4 任务与异常工作台

建议采用队列式设计，便于用户理解当前系统“正在做什么”“哪里出了问题”“哪些需要稍后处理”。

## 11. 非功能要求

### 11.1 性能目标

产品方向上应按以下目标设计：

- 支持超大量文件管理，目标上限为上亿级文件。
- 支持超大单体文件，目标上限为数 TB。
- 支持超大总容量，目标上限为百 TB 级以上。
- 文件列表、筛选、状态查询和任务查看应保持高性能响应。

说明：

上述是产品与架构目标，不代表首版上线即完成所有极限场景验证，但首版架构不得把后续规模上限直接锁死。

### 11.2 可靠性目标

- 导入、复制、同步任务必须支持断点续传。
- 关键任务必须支持结果校验。
- 异常状态必须可追踪、可恢复、可人工介入。

### 11.3 可扩展性目标

- 新增存储端不应重做上层任务系统。
- 新增自动归档规则不应推翻现有导入模型。
- 媒体元数据能力应支持逐步扩展更多字段和格式。

## 13. 关键风险与设计提醒

- 以相对路径、名称、大小、类型为主的同文件判定规则实现成本低于全量内容哈希，但在复杂迁移和历史数据整理场景中，需要特别注意误归并和漏归并风险。
- 首版要求替代大量传统文件夹操作，会使目录语义、资产语义和跨端任务语义强耦合，产品和数据模型必须从一开始统一设计。
- 115 网盘及后续主流网盘接入高度依赖外部能力，需尽早确认接入策略、稳定性边界与异常恢复方案。
- “上亿文件、百 TB、数 TB 单文件”属于高强度目标，后续架构设计必须提前把索引、缓存、分页、任务恢复和数据库选型作为核心议题。

## 14. 成功判断标准

若首版上线后满足以下结果，可认为产品方向成立：

- 用户能够在系统中统一查看和管理多端文件资产。
- 用户能清晰回答某个文件当前在哪些端点上存在。
- 用户能够在系统中完成原本依赖 Windows 文件夹的大部分常见操作。
- 用户能够稳定完成从移动硬盘/U 盘到多个目标端的入库任务。
- 用户能够对跨端同步任务建立信任，不再担心漏拷、错拷和中断重来。
- 用户愿意把本系统作为日常文件管理主入口，而不是偶尔打开的辅助工具。
