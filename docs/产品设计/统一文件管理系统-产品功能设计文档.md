# 统一文件管理系统产品功能设计文档

## 文档说明

- 更新时间：2026-04-11
- 对照范围：`client/src`、`client/src-tauri/src`
- 文档目标：将产品文档刷新为“当前客户端实际已经实现的内容”，不再混写未来规划口径

## 1. 当前产品定位

当前客户端是一个面向桌面端的统一文件管理工作台原型，重点验证以下三件事：

1. 用一个多标签工作区承接文件中心、导入中心、任务中心、异常中心、存储节点和设置页。
2. 用本地状态和本地数据源把“文件操作 -> 任务 -> 异常 -> 通知 -> 回跳处理”的闭环跑通。
3. 用 Tauri 命令层验证存储节点管理页面的真实桌面交互能力。

当前实现更接近“高保真业务原型 + 局部真实后端闭环”，不是完整生产版客户端。它已经具备大量真实的产品交互，其中存储节点页和文件中心主资产链路已经大范围接入中心服务真实接口，但仍保留浏览器 fallback、标签本地库和少量 Tauri 桥接能力作为过渡。

## 2. 当前已实现的一级信息架构

### 2.1 左侧主导航

左侧主导航当前固定包含 5 个一级页面：

- 文件中心
- 任务中心
- 异常中心
- 存储节点
- 设置

### 2.2 页头入口

页头当前还有两个独立入口：

- 导入中心入口：显示设备接入摘要，点击后打开“导入中心”工作区标签
- 通知中心入口：显示未消费通知角标，点击后以抽屉形式打开“通知中心”

这意味着导入中心和通知中心不在左侧主导航中，而是通过页头信号进入。

### 2.3 顶层多标签工作区

所有一级页面都运行在顶层多标签工作区内。当前支持：

- 打开新标签
- 关闭当前、关闭其他、关闭左侧、关闭右侧
- 重新打开最近关闭的标签
- 标签拖拽排序
- 标签右键菜单刷新
- 页面级保活，不销毁已挂载页面的筛选与草稿状态

## 3. 当前核心业务对象

### 3.1 资产库

客户端内置 3 个资产库：

- 商业摄影资产库
- 视频工作流资产库
- 家庭照片资产库

当前资产库主要用于：

- 文件中心按库切换
- 导入中心设备会话挂靠
- 任务、异常、通知按库归属
- 标签作用域控制
- 存储节点挂载目录归属

### 3.2 统一资产

文件中心中的统一资产当前由 `fileCenterApi` 作为客户端适配层承接，核心资产数据优先来自中心服务，包含：

- 文件与目录
- 生命周期状态：`ACTIVE`、`PENDING_DELETE`
- 端点副本状态
- 星级、色标、标签
- 元数据行
- 风险标签、徽标、最近任务摘要

### 3.3 存储端与副本

当前产品语义里同时存在两套相关概念：

- 文件中心中的副本端点状态：用于表现单个资产在本地、NAS、网盘等端点上的同步情况
- 存储节点页面中的挂载目录 / NAS / 网盘配置：用于表现客户端如何接入实际存储来源

两者有关联，但当前实现还没有真正打通为统一后端模型。

### 3.4 导入设备会话

导入中心当前按“设备会话”组织工作，而不是按导入任务列表组织。每个设备会话包含：

- 设备信息与挂载路径
- 预检草稿
- 文件清单
- 可选目标端
- 最近导入报告
- 关联异常

### 3.5 任务

任务中心当前分为两类：

- 传输任务：导入、同步，链路类型含复制 / 上传 / 下载
- 其他任务：扫描、元数据解析、校验、删除清理

任务支持主任务与子项两层结构，且支持在任务中心内修改优先级和运行状态。

### 3.6 异常

异常中心当前统一承接以下来源：

- 传输任务
- 其他任务
- 文件中心
- 存储节点
- 系统治理

异常具备：

- 分类
- 严重级别
- 问题性质（阻塞 / 风险）
- 处理能力开关
- 来源上下文
- 影响范围
- 处理历史

### 3.7 通知

通知中心当前只承接两类通知：

- 处置类通知 `ACTION_REQUIRED`
- 提醒类通知 `REMINDER`

通知支持跳转到：

- 异常中心
- 任务中心
- 文件中心
- 存储节点
- 导入中心

## 5. 当前关键交互闭环

### 5.1 文件中心到任务中心

- 同步资产会创建传输任务
- 删除资产或删除端点副本会创建删除清理类任务
- 删除冲突时可先跳转任务中心查看阻塞任务

### 5.2 导入中心到任务中心

- 提交导入会把设备会话切到“导入中”
- 同时写入导入报告快照
- 同时创建一条传输任务及其子项

### 5.3 任务中心到异常中心 / 文件中心

- 任务异常浮窗可以直达异常中心
- 任务详情可以直达文件中心定位相关资产

### 5.4 异常中心到原场景

- 异常详情可打开任务中心、文件中心、存储节点
- 从通知中心进入异常中心时会带上下文筛选

### 5.5 通知中心到各工作区

- 通知可按目标类型直接跳入对应工作区
- 跳转时会联动任务筛选、异常上下文或文件定位

## 6. 当前数据与持久化边界

### 6.1 文件中心与标签管理

文件中心与标签管理当前不是同一条数据链路：

- 文件中心的资产、目录、详情、扫描、新建目录、上传、星级 / 色标、资产删除优先走中心服务与 PostgreSQL
- 标签管理、标签建议、端点同步与删除端点副本仍主要依赖 `sql.js` 与浏览器本地存储

### 6.2 全局客户端状态

任务、异常、通知、导入中心、设置等状态主要来自 `clientState`，也通过 `localStorage` 保存，但并不是所有字段都保留；部分任务和导入数据在重新加载时会重新用种子数据覆盖。

### 6.3 存储节点

存储节点当前优先通过中心服务接口读取和写入真实数据；浏览器环境仍保留本地 fallback，Tauri 环境仅保留本地目录选择等必要 OS 桥接能力。

## 7. 当前未实现或仅为占位的内容

以下内容当前文档必须明确标记为“未实现”或“未完全打通”，不能再按生产能力表述：

- 真实用户体系、权限体系、登录体系

## 8. 当前版本结论

截至 2026-04-10，客户端已经形成一套完整的桌面端业务工作台骨架：

- 文件中心负责资产视图和直接操作
- 导入中心负责设备会话与提交前编排
- 任务中心负责运行期跟踪
- 异常中心负责统一治理
- 通知中心负责轻提醒与跳转
- 存储节点负责接入配置与扫描验证
- 设置页负责默认策略与工作台预览

当前最重要的产品结论不是“系统已生产化”，而是“客户端主要交互闭环已经跑通，且文件中心 / 存储节点已经出现真实后端主链路，文档必须按这条混合实现边界来写”。
