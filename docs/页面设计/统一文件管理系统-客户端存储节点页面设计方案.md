# 统一文件管理系统-客户端存储节点页面设计方案

## 文档信息

- 文档名称：统一文件管理系统-客户端存储节点页面设计方案
- 文档用途：记录当前客户端“存储节点”页面的实际实现
- 适用范围：Windows 桌面客户端首版原型
- 当前状态：已按当前实现刷新
- 更新时间：2026-04-10

## 1. 页面定位

当前“存储节点”不是单纯的状态看板，而是一个存储配置工作台。页面围绕“挂载文件夹”展开，并补充 NAS 与网盘配置；其中读写、测试、删除等主路径已经大范围切到中心服务，浏览目录仍保留 Tauri 桥接，本地 fallback 也还在。

页面重点完成：

- 管理挂载文件夹
- 管理 NAS 主机
- 管理网盘接入
- 发起扫描与连接测试
- 查看扫描历史

补充说明（截至 2026-04-11）：

- “发起扫描”现在已经不是同步执行，而是会创建真实扫描作业
- 这些作业由中心服务统一调度，后续由任务中心切片接入展示与操作

## 2. 页面入口

- 存储节点是左侧导航一级页面
- 已纳入顶层多标签工作区

## 3. 当前子页结构

页面当前分为 3 个子页：

1. 挂载管理
2. NAS 管理
3. 网盘管理

## 4. 顶部工具区

当前顶部支持：

- 子页切换
- 类型筛选
- 状态筛选
- 搜索
- 新增挂载文件夹 / 新增 NAS / 新增网盘

搜索提示会随子页变化：

- 挂载页：搜索名称、路径、资产库
- NAS 页：搜索名称、地址、账号
- 网盘页：搜索名称、目录、账号别名

通用字段包括：

## 9. 当前数据来源

当前页面使用 `storageNodesApi`，具备三层路径：

- 真实 center API
- 浏览器 fallback
- Tauri Rust 命令桥接

如果运行在 Tauri 环境，会调用本地命令：

- 加载仪表盘
- 保存配置
- 扫描
- 连接测试
- 更新心跳
- 读取历史
- 删除
- 浏览本地目录

其中浏览本地目录仍然是 Tauri 独占桥接，其余主路径优先走中心服务。

## 10. 当前与其它页面的关系

- 挂载扫描会创建真实扫描作业，后续任务中心应消费该作业并展示在“其他任务”中
- 挂载异常可跳转异常中心
- 目标端查看可回到导入中心或文件中心上下文

## 11. 当前边界

- 没有真实磁盘扫描执行器完全接管
- 没有真实 NAS / 网盘鉴权全链路完全收口
- 没有真实挂载执行器完全接管
- 心跳策略更新接口与中心服务路由仍在收口中
- 连接测试结果已大范围由中心服务回写，但浏览器 fallback 下仍保留模拟结果

## 12. 当前结论

截至 2026-04-11，存储节点页已经形成结构完整的配置工作台，是当前客户端少数已经大范围接入真实中心服务的业务页面。当前挂载扫描已切入真实作业体系；浏览本地目录等桌面壳层能力仍保留 Tauri 桥接，任务中心消费层仍待后续切片完成。
