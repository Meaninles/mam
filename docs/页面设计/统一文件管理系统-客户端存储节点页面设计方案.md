# 统一文件管理系统-客户端存储节点页面设计方案

## 文档信息

- 文档名称：统一文件管理系统-客户端存储节点页面设计方案
- 文档用途：用于客户端“存储节点”页面的产品、交互与界面实现对齐
- 适用范围：Windows 桌面客户端首版
- 当前阶段：页面设计方案 / 实现前规划稿

## 1. 设计背景

根据现有产品与技术文档，系统需要支持本机磁盘、移动硬盘/U 盘、SMB/NAS、115 网盘等多种存储端点，并提供统一的接入、配置、扫描、连接测试、心跳计划、异常联动与任务协同能力。

客户端中的“存储节点”页面不是简单的配置页，而是面向日常管理的节点工作台。但该页面必须坚持“节点列表为主”，避免把首屏做成监控大盘、设置中心首页或复杂详情页。

## 2. 本页面的核心设计原则

### 2.1 列表主导

- 页面进入后的最主要展示内容必须是存储节点列表。
- 用户的主要浏览、筛选、选择、批量操作、单节点操作都围绕列表完成。
- 详细配置、扫描历史、鉴权信息、心跳设置、P2 能力均采用隐藏弹出式设计，不在首屏喧宾夺主。

### 2.2 与当前客户端设计保持一致

所有 UI、UX 与设计风格必须与当前客户端保持一致，不单独创建新的视觉体系。

应延续的现有设计语言包括：

- 颜色体系：沿用当前客户端浅色/深色双主题、蓝色主强调色、成功/警告/危险状态色
- 字体体系：沿用当前客户端正文与数字字体组合
- 组件风格：圆角卡片、玻璃态背景、紧凑型列表、轻量按钮、抽屉式 Sheet、状态胶囊、进度条
- 交互风格：工具栏 + 列表 + 隐藏弹层，避免重型仪表盘式页面
- 状态反馈：延续现有 `TonePill`、`ProgressBar`、`FeedbackBanner`、`Sheet` 这一套组件语义

### 2.3 类型明确

- 节点必须明确区分：本地节点、NAS 节点、网盘节点。
- 网盘节点需要有厂商概念，当前仅实现 `115`，但页面结构必须预留后续扩展位。
- 不同类型节点使用不同配置字段和不同交互提示。

### 2.4 操作就近发生

- 单节点管理应主要通过节点行右侧图标按钮完成。
- 复杂配置、历史信息、心跳设置、鉴权设置采用隐藏弹出方式展开。
- 用户不应频繁离开列表页去做管理动作。

### 2.5 耗时操作必须有充分反馈

- 扫描、连接测试、批量扫描、批量连接测试等操作必须提供清晰的 loading、进行中与结果反馈。
- 反馈应优先贴近节点行和列表上方工具条呈现，不打断主要浏览流程。

## 3. 页面职责

“存储节点”页面负责以下能力：

- 展示全部已配置存储节点
- 筛选、搜索、选择节点
- 新增、编辑、启用、停用、删除节点
- 对单节点或多节点发起扫描
- 对单节点或多节点发起连接测试
- 配置 NAS / 网盘访问鉴权信息
- 配置 NAS / 网盘心跳周期
- 查看节点的最近扫描时间、扫描结果、连接状态摘要
- 跳转或联动查看与节点相关的任务和异常
- 为后续 P2 能力预留入口与占位

## 4. 页面信息架构

页面结构采用“轻工具栏 + 主列表 + 隐藏弹出层”的形式。

### 4.1 顶部轻工具栏

左侧：

- 页面标题：`存储节点`
- 页面说明：`管理本地、NAS 与网盘节点，执行扫描、连接测试与心跳配置`

右侧：

- 类型筛选
- 状态筛选
- 搜索框
- `新增节点`
- `批量扫描`
- `批量连接测试`

### 4.2 主体区域

主体区域以节点列表为绝对主角，不常驻右侧详情栏，不使用大面积概览卡。

### 4.3 隐藏弹出层

通过以下隐藏方式承接复杂信息：

- 右侧抽屉：新增节点 / 编辑节点 / 鉴权配置
- 小弹层：心跳周期设置 / 批量设置心跳
- 行内结果浮层：连接测试结果
- 弹窗：扫描历史 / P2 占位说明 / 批量结果汇总
- 更多菜单：低频操作和次级入口

## 5. 节点列表设计

### 5.1 列表目标

节点列表是本页面的主工作区。用户在这里完成：

- 识别节点
- 判断节点当前是否可管理
- 选择节点
- 发起单节点操作
- 发起多节点批量操作

### 5.2 推荐列定义

从左到右建议采用以下列：

1. 选择框
2. 节点名称
3. 节点类型
4. 地址/路径
5. 扫描状态
6. 最近扫描时间
7. 心跳周期
8. 容量/可用空间摘要
9. 操作区

补充说明：

- 节点名称列可附带次级标签：`只读`、`可写`、`已绑定 2 个资产库`、`115`
- 地址列展示实际路径、NAS 地址或网盘目录
- 扫描状态列只显示最关键结果，不展开长说明
- 容量列使用文本 + 轻量进度条组合

### 5.3 行内信息优先级

每一行应先保证用户能快速看明白：

1. 这是什么节点
2. 它在哪
3. 最近是否扫过
4. 能不能继续用
5. 下一步可以做什么

### 5.4 行右侧操作区

节点管理主要通过节点行右侧图标按钮实现，推荐顺序如下：

- `连接测试`
- `立即扫描`
- `编辑`
- `更多`

其中：

- `连接测试` 使用雷达/探测类图标
- `立即扫描` 使用扫描/刷新类图标
- `编辑` 使用铅笔图标
- `更多` 使用省略号图标

`更多` 菜单内包含：

- 启用
- 停用
- 设置心跳
- 鉴权设置
- 查看扫描历史
- 查看相关任务
- 查看相关异常
- 删除

## 6. 多选与批量操作设计

### 6.1 选择模型

页面必须支持“可选择性的多选批量操作”，不能只支持单行操作和全量操作。

具体规则：

- 列表左侧提供逐行复选框
- 表头提供总复选框
- 支持任意勾选若干节点后执行批量操作
- 不强制用户“全选全部节点”

建议提供以下选择辅助能力：

- 全选当前页
- 全选当前筛选结果
- 清空选择

### 6.2 批量工具条

当用户选中一个或多个节点后，列表上方出现选择态工具条，显示：

- `已选择 X 个节点`
- 批量扫描
- 批量连接测试
- 批量启用
- 批量停用
- 批量设置心跳
- 清空选择

批量工具条应轻量呈现，不遮挡列表主体，不独占过多高度。

### 6.3 批量操作边界

适合批量执行的操作：

- 扫描
- 连接测试
- 启用
- 停用
- 设置心跳周期

不建议做成批量直接编辑的操作：

- 修改节点名称
- 修改地址/路径
- 修改 NAS 鉴权信息
- 修改网盘鉴权信息

这些仍保留为单节点配置动作。

## 7. 节点类型与配置模型

### 7.1 本地节点

字段建议：

- 节点名称
- 根路径
- 挂载模式：只读 / 可写
- 是否允许扫描
- 心跳周期
- 备注

交互特征：

- 不需要鉴权配置
- 连接测试更偏向路径可访问性检查

### 7.2 NAS 节点

字段建议：

- 节点名称
- 协议类型
- 主机/IP
- 共享目录
- 用户名
- 密码
- 挂载模式：只读 / 可写
- 心跳周期
- 备注

交互特征：

- 支持连接测试
- 支持保存鉴权信息
- 支持心跳探测

### 7.3 网盘节点

字段建议：

- 节点名称
- 厂商
- 账号别名
- 远端目录
- 鉴权方式
- 令牌状态
- 挂载模式：只读 / 可写
- 心跳周期
- 备注

交互特征：

- 当前厂商下拉仅提供 `115`
- 页面结构必须保留后续网盘厂商扩展位
- 支持连接测试
- 支持鉴权配置
- 支持心跳探测

## 8. 新增与编辑节点交互

新增和编辑节点统一采用隐藏式右侧抽屉，不使用独立页面。

建议采用三段式表单组织：

### 8.1 第一段：基础信息

- 节点名称
- 节点类型
- 节点备注

### 8.2 第二段：类型专属配置

根据节点类型动态切换表单：

- 本地：路径
- NAS：主机、共享目录、用户名、密码
- 网盘：厂商、账号别名、远端目录、鉴权信息

### 8.3 第三段：管理选项

- 挂载模式
- 心跳周期
- 连接测试按钮
- 保存按钮

表单要求：

- 错误信息就近显示在字段下方
- 不允许仅靠 placeholder 表达字段含义
- 敏感字段独立分组
- 连接测试结果显示在保存区附近

## 9. 扫描设计

### 9.1 扫描的页面语义

本页面中的扫描是管理动作，不是单纯展示状态。

扫描主要用于：

- 初始化建库
- 对账
- 恢复
- 异常修复
- 外部变更回收

### 9.2 扫描入口

页面需要支持：

- 单节点立即扫描
- 多节点批量扫描
- 查看最近扫描时间
- 查看最近扫描结果

### 9.3 扫描状态

列表中建议支持以下状态：

- 未扫描
- 最近扫描成功
- 最近扫描失败
- 扫描中
- 等待队列
- 上次中断

### 9.4 扫描反馈

扫描是耗时操作，必须提供充分提示：

- 点击“立即扫描”后，该行按钮进入 loading 状态
- 扫描状态列立即变为“扫描中”
- 最近扫描时间列显示“正在执行”
- 页面顶部出现轻量提示：`扫描已开始，可继续浏览其他节点`
- 批量扫描时，顶部提示展示汇总结果，如 `已为 3 个节点创建扫描任务`

## 10. 连接测试设计

### 10.1 测试入口

连接测试支持：

- 单节点连接测试
- 多节点批量连接测试
- 表单内连接测试

### 10.2 结果展示原则

测试结果不能只显示“成功”或“失败”，应尽量拆成分项：

- 可达性
- 鉴权状态
- 读权限
- 写权限
- 目标目录可访问

### 10.3 测试反馈

连接测试是耗时操作，必须提供 loading：

- 按钮文案改为 `测试中...`
- 图标按钮显示转圈
- 结果区域显示骨架或占位说明
- 失败时提供下一步建议：`重试`、`检查配置`、`重新鉴权`

## 11. 心跳周期设计

### 11.1 心跳适用范围

心跳重点服务于 NAS 和网盘节点，本地节点也可配置，但默认建议为 `从不`。

### 11.2 心跳选项

客户端首版固定提供以下选项：

- 从不
- 每周（深夜）
- 每日（深夜）
- 每小时

### 11.3 心跳入口

入口形式：

- 单节点行 `更多 -> 设置心跳`
- 多选后 `批量设置心跳`

### 11.4 心跳展示

列表中显示当前心跳周期。

隐藏弹层中展示：

- 当前周期
- 下次预计执行时间
- 最近一次结果
- 最近连续失败次数

## 12. 隐藏弹出式设计清单

为避免喧宾夺主，下列内容不在页面主体常驻显示：

- 节点新增抽屉
- 节点编辑抽屉
- NAS / 网盘鉴权配置抽屉
- 心跳设置弹层
- 连接测试结果浮层
- 扫描历史弹窗
- P2 功能占位弹窗

## 13. 耗时操作统一 UX 规则

以下操作统一视为耗时操作：

- 连接测试
- 单节点扫描
- 多节点批量扫描
- 多节点批量连接测试

统一规则：

- 必须出现明确 loading 态
- 不允许点击后无反馈
- 优先在节点行和工具条附近反馈
- 完成后给出成功/失败汇总提示
- 错误提示必须可恢复，不只说“失败”

## 14. P2 功能占位

P2 能力在客户端中先设计占位，但不作为首版主内容常驻展示。

建议占位项：

- 扫描历史详情
- 节点模板 / 复制配置
- 容量趋势
- 更多网盘厂商
- 高级心跳策略

占位形式：

- `更多` 菜单中的灰态入口
- `即将支持` 弹层说明

## 15. 与任务中心、异常中心的联动

本页面不承担完整任务与异常展示，但要提供就近入口：

- 行 `更多 -> 查看相关任务`
- 行 `更多 -> 查看相关异常`
- 若节点存在风险，可在节点名称下或状态列用轻量标签提示：
  - 扫描失败
  - 连接异常
  - 鉴权异常
  - 空间风险

## 16. 推荐交互状态清单

节点行至少应覆盖以下前端状态：

- 正常
- 被选中
- 扫描中
- 连接测试中
- 最近扫描失败
- 鉴权异常
- 已停用
- 只读
- 可写

弹层/抽屉至少应覆盖以下状态：

- 初始态
- 加载中
- 校验失败
- 测试成功
- 测试失败
- 保存成功

## 17. 结论

客户端“存储节点”页面的正确方向应为：

- 以节点列表为主工作区
- 以节点行右侧图标按钮为主要管理入口
- 以多选批量工具条支持选择性批量操作
- 以扫描、连接测试、心跳配置为核心节点管理能力
- 以抽屉、弹层、菜单承载复杂设置
- 在所有 UI、UX 与视觉风格上严格保持与当前客户端实现一致

该设计既满足现有产品与架构文档要求，也满足后续对更多节点类型、更多网盘厂商与更复杂心跳策略的扩展需要。
