# 统一文件管理系统-客户端顶层多标签工作区设计方案

## 文档信息

- 文档名称：统一文件管理系统-客户端顶层多标签工作区设计方案
- 文档用途：记录当前客户端顶层工作区的实际行为基线
- 适用范围：Windows 桌面客户端前端实现
- 当前状态：已与当前代码实现对齐
- 更新时间：2026-04-08

## 1. 页面定位

客户端顶层多标签工作区是一级页面之上的壳层，不是新的业务页。它负责承载一级页面的打开、切换、排序、关闭与恢复。

当前纳入工作区的一级页面有：

1. 文件中心
2. 导入中心
3. 任务中心
4. 异常中心
5. 存储节点
6. 设置

## 2. 当前已实现规则

### 2.1 打开规则

- 文件中心、任务中心、异常中心、存储节点、设置通过左侧导航打开
- 导入中心通过页头动态导入入口打开
- 同一一级页面不会重复打开多个标签

### 2.2 切换规则

- 点击标签切换页面
- 点击已存在的导航项会直接聚焦对应标签
- 活跃页面状态会在标签上高亮显示

### 2.3 关闭规则

- 标签无显式 `X` 按钮
- 关闭动作通过右键菜单触发
- 支持关闭当前、关闭其他、关闭左侧、关闭右侧
- 关闭当前标签后，优先激活右侧相邻标签，其次激活左侧标签

### 2.4 排序规则

- 支持拖拽排序
- 支持右键菜单“向左移动”“向右移动”

### 2.5 恢复规则

- 支持“重新打开刚关闭的标签”
- 恢复来源为最近关闭栈
- 只支持会话内恢复，不支持应用重启后恢复

### 2.6 壳层系统状态指示

- 顶层页头已新增系统状态灯
- 位置位于右上区域、导入入口左侧
- 状态灯不属于新的一级页面，也不进入顶层标签体系
- 状态灯仅以红 / 黄 / 绿三种视觉状态展示系统运行情况
- 鼠标悬停时显示中心服务与本地执行器的状态描述
- 状态判断规则为：
  - 中心服务异常：红灯
  - 中心服务正常、本地执行器异常：黄灯
  - 中心服务和本地执行器都正常：绿灯

## 3. 不纳入顶层标签体系的内容

以下内容当前不进入顶层工作区：

- 通知中心右侧侧窗
- 文件详情抽屉
- 任务详情抽屉
- 存储节点新增 / 编辑抽屉
- 导入预检详情、导入报告快照
- 标签管理内部的弹窗与确认框

## 4. 当前上下文保活

当前工作区切换会保留页面上下文，尤其包括：

- 搜索词
- 筛选项
- 分页
- 选中项
- 草稿
- 当前子页或子标签状态

这也是当前工作区壳层最重要的价值之一。

## 5. 当前实现边界

- 不支持同页多实例
- 不支持跨重启恢复标签布局
- 不支持标签固定
- 不支持标签分组
- 不支持从通知中心或弹层直接生成独立工作区标签

## 6. 当前结论

截至 2026-04-08，顶层多标签工作区已经是客户端的稳定壳层基础设施。后续所有一级页面设计都应默认遵守本文规则，避免再把抽屉、侧窗或页内子视图误写成顶层标签页。
