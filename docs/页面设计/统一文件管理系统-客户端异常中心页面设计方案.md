# 统一文件管理系统-客户端异常中心页面设计方案

## 文档说明

- 更新时间：2026-04-13
- 对照代码：`client/src/pages/IssuesPage.tsx`、`client/src/App.tsx`、`client/src/data.ts`
- 目标：将异常中心文档刷新为当前真实实现

## 1. 页面定位

异常中心是当前客户端统一治理入口，用来收拢：

- 传输任务异常
- 其他任务异常
- 文件中心风险
- 存储节点风险
- 系统治理问题

它的职责不是执行所有强动作，而是：

- 聚合
- 筛选
- 轻量处置
- 跳回原场景

## 2. 当前入口与上下文聚焦

### 2.1 入口

当前可以从以下位置进入异常中心：

- 左侧主导航
- 通知中心
- 存储节点页面
- 导入中心预检摘要或结果摘要

### 2.2 上下文聚焦

异常中心当前支持通过 `focusRequest` 注入上下文，自动完成：

- 来源域筛选
- 资产库筛选
- 任务 / 路径 / 端点上下文过滤
- 自动选中指定异常
- 自动打开详情抽屉

页面顶部还会显示上下文横幅，并支持“清除筛选”。

## 3. 当前页面结构

异常中心当前包含：

- 工具条层
- 选中态批量工具条
- 上下文横幅层
- 列表主体层
- 详情抽屉层

## 4. 当前筛选与搜索

### 4.1 一级分类筛选

当前支持：

- 全部
- 冲突
- 传输
- 校验
- 节点与权限
- 容量与资源
- 扫描与解析
- 清理与治理

### 4.2 问题性质筛选

当前支持：

- 全部问题
- 阻塞型异常
- 提醒型风险

### 4.3 来源域筛选

当前支持：

- 全部来源
- 传输任务
- 其他任务
- 文件中心
- 存储节点
- 系统治理

### 4.4 资产库筛选

当前支持按资产库名过滤。

### 4.5 状态筛选

当前支持：

- 全部
- 待处理
- 待确认
- 处理中
- 已忽略
- 已解决
- 已归档

### 4.6 严重级别筛选

当前支持：

- 全部级别
- 高优先级
- 需尽快处理
- 信息

### 4.7 搜索

当前搜索范围包括：

- 标题
- 摘要
- 类型
- 分类
- 资产

- 详情
- 建议动作
- 任务标题
- 子项标题
- 端点标签
- 路径
- 来源摘要

### 4.8 排序

当前支持：

- 默认排序
- 最近更新时间
- 严重级别
- 名称

## 5. 当前列表设计

## 6. 当前动作体系

### 6.1 单项动作

根据异常能力开关与当前状态，页面当前可能出现：

- 重试
- 标记已确认
- 忽略
- 归档
- 打开设置页
- 打开任务中心
- 打开文件中心
- 打开存储节点

补充：`打开设置页` 当前由异常中心直接联动到设置页“依赖服务”子页（`dependency-services`）。

### 6.2 批量动作

对选中异常，当前会按“所有项都可执行”原则生成批量动作：

- 批量重试
- 批量标记已确认
- 批量忽略
- 批量归档

### 6.3 历史清理

当前支持：

- 清理当前筛选下的历史项
- 清理选中的历史项

当前被视为历史项的状态包括：

- 已忽略
- 已解决
- 已归档

### 6.4 云端治理入口策略（已实现）

异常中心当前已经在详情抽屉与更多菜单中落地云端入口策略：

- 识别为 `115 鉴权问题` 时，推荐入口为 `打开存储节点`
- 识别为 `CloudDrive2 网关问题` 或 `aria2 下载器问题` 时，推荐入口为 `打开设置页`
- 识别为 `云端路径或目录问题` 或 `传输执行问题` 时，推荐入口为 `打开任务中心`

其中 `CloudDrive2` / `aria2` 场景会显示并支持“打开设置页”；`115 鉴权` 场景保留“打开存储节点”作为治理主入口。

## 7. 当前详情抽屉

详情抽屉当前由以下区块组成：

### 7.1 概览区

展示：

- 异常类型
- 一级分类
- 问题性质
- 严重级别
- 当前状态
- 所属资产库
- 首次发现时间
- 最近更新时间
- 当前建议动作

### 7.2 来源上下文区

展示：

- 来源域
- 来源摘要
- 执行链路
- 关联任务
- 关联任务子项
- 关联端点
- 关联路径
- 影响对象

补充说明：执行链路来自 `issue.source.routeLabel`，用于在异常详情中直接还原“本地/NAS/网盘 -> 目标端”的真实传输路径。

### 7.3 云端治理区（已实现）

当异常被识别为云端相关问题时，详情抽屉会新增“云端治理”区块，包含：

- 云端问题归因
- 推荐处理入口

当前归因结果包括：

- `115 鉴权问题`
- `CloudDrive2 网关问题`
- `aria2 下载器问题`
- `云端路径或目录问题`
- `传输执行问题`

### 7.4 影响范围区

展示：

- 影响资产数
- 影响副本数
- 影响目录数
- 影响端点数
- 是否阻塞正式状态提交
- 是否阻塞任务继续执行

### 7.5 建议与详情区

展示：

- 处理建议
- 详细说明

### 7.6 处理历史区

展示：

- 历史动作
- 处理结果
- 操作者
- 时间

## 8. 当前与其他页面的联动

### 8.1 与任务中心

- 可按任务聚焦异常
- 详情抽屉可打开任务中心

### 8.2 与文件中心

- 可按文件或路径聚焦异常
- 详情抽屉可打开文件中心

### 8.3 与存储节点

- 可按端点或路径聚焦异常
- 详情抽屉可打开存储节点

补充：`115 鉴权` 相关异常在治理路径上默认引导到存储节点页处理鉴权与端点连通性。

### 8.4 与通知中心

- 通知中心可把用户带到异常中心，并注入上下文

### 8.5 与设置页（依赖服务）

- `CloudDrive2 网关问题`、`aria2 下载器问题` 可直接从异常详情或更多菜单打开设置页
- 打开后默认定位到“依赖服务”子页，用于处理 `CloudDrive2/aria2` 运行态与连通性问题

## 9. 当前数据来源

异常中心数据当前来自：

- 中心服务 `GET /api/issues`
- 中心服务 `POST /api/issues/actions`
- 中心服务 `POST /api/issues/history/clear`
- 客户端资产库列表

当前异常中心已经改接真实 center API；`PersistedState.issueRecords` 不再作为正式数据源。

补充说明：异常对象当前已经由中心服务按稳定组合键去重复用，同一问题重复出现时会回并到历史异常对象，并通过发生次数累计展示。

## 10. 当前实现边界

以下内容当前没有实现：

- 真正的异常评论 / 协作
- 自动化规则编辑器
- SLA / 责任人体系
- 真正的时间轴回放
- 批量导出
- 嵌套树形异常组织

## 11. 当前版本结论

截至 2026-04-13，异常中心已经具备统一治理入口应有的基本能力：

- 多维筛选
- 上下文聚焦
- 真实轻量处置动作
- 详情抽屉
- 云端问题归因与推荐处理入口
- 来源上下文中的执行链路展示
- CloudDrive2/aria2 问题直达设置页（依赖服务）
- 115 鉴权问题直达存储节点
- 历史清理
- 与任务 / 文件 / 存储节点的联动
- 同一异常的发生次数展示
- 与通知中心的真实通知联动

当前应按“已接入中心服务异常域的真实治理工作台”来描述。通知中心已经接入真实通知事件，但通知页仍保持轻提醒与跳转定位，不扩展为治理页面。
