# 统一文件管理系统-客户端导入中心页面设计方案

## 文档说明

- 更新时间：2026-04-08
- 对照代码：`client/src/pages/ImportCenterPage.tsx`、`client/src/lib/importCenter.ts`、`client/src/App.tsx`、`client/src/data.ts`
- 目标：将导入中心文档刷新为当前实际实现

## 1. 页面定位

导入中心是当前客户端面向“现场介质接入 -> 导入前编排 -> 提交导入”的专用工作台。它当前承担的职责是：

- 管理设备会话
- 管理导入草稿
- 选择目标端
- 查看预检摘要
- 检查文件级状态
- 提交导入并转交任务中心

当前导入中心不是任务监控页，而是“提交前工作台”。

## 2. 当前入口与导航关系

### 2.1 入口位置

导入中心当前不在左侧主导航中，而是通过页头“导入入口芯片”进入。

页头芯片会根据设备会话状态显示：

- 已插入设备数量
- 需要关注的设备数量
- 当前视觉 tone

### 2.2 与顶层多标签工作区关系

导入中心打开后会成为一个顶层工作区标签，与文件中心、任务中心等并列存在。

### 2.3 当前未实现的内容

虽然 `importCenter.ts` 里已经预留了 `ImportCenterTab` 类型，但页面当前并没有实现页内多标签。

当前界面是：

- 左侧设备会话列表
- 右侧当前选中会话工作区

## 3. 当前页面布局

导入中心当前由以下部分组成：

### 3.1 顶部工具条

包含：

- 会话状态筛选
- 搜索设备名称或地址
- 排序方式
- 刷新设备
- 最近摘要

### 3.2 左侧设备会话列表

每个设备行展示：

- 设备名称
- 挂载路径
- 可用空间
- 快捷操作按钮

快捷操作当前包括：

- 打开会话
- 重新扫描
- 查看预检
- 查看详情

### 3.3 右侧当前会话工作区

按顺序展示：

- 来源摘要卡
- 目标编排卡
- 预检摘要卡
- 文件清单卡
- 结果摘要卡（如果有最近报告）
- 提交区

## 4. 当前会话模型

当前导入中心围绕“设备会话”展开。每个会话当前包含：

- 设备类型
- 挂载路径
- 资产库归属
- 扫描状态
- 会话状态
- 预检草稿
- 最近导入报告
- 关联异常
- 可用目标端列表

### 4.1 当前会话状态

页面实际会出现的会话状态包括：

- 待识别
- 扫描中
- 可导入
- 导入中
- 部分完成
- 异常待处理
- 已拔出

### 4.2 当前文件级状态

文件清单中会出现的状态包括：

- 待导入
- 已排队
- 传输中
- 校验中
- 已完成
- 失败
- 冲突
- 已跳过

## 5. 当前设备列表交互

### 5.1 状态筛选

当前支持：

- 全部
- 待扫描
- 可导入
- 导入中
- 异常

### 5.2 搜索

当前搜索范围包括：

- 设备名称
- 设备类型
- 路径
- 描述
- 会话状态

### 5.3 排序

当前支持：

- 最近接入
- 状态优先
- 名称
- 剩余空间

## 6. 当前会话工作区设计

### 6.1 来源摘要卡

展示：

- 设备名称
- 会话状态
- 来源路径
- 容量 / 可用空间
- 文件数 / 文件夹数
- 最近扫描时间

并提供：

- 重新扫描
- 重新预检
- 打开任务中心

### 6.2 目标编排卡

展示当前设备可用目标端，并对每个目标端显示：

- 目标端名称
- 目标端类型
- 可用空间或状态标签
- 当前覆盖范围（全部 / 部分 / 未覆盖）

当前操作包括：

- 应用到全部文件
- 全部取消
- 查看目标端

### 6.3 预检摘要卡

展示：

- 通过数
- 风险数
- 阻塞数
- 前四条预检项

并支持：

- 查看预检详情
- 打开异常中心

### 6.4 文件清单卡

支持：

- 文件状态筛选
- 文件搜索
- 文件级目标端勾选

文件状态筛选当前支持：

- 全部
- 可导入
- 已导入
- 冲突
- 异常

### 6.5 结果摘要卡

只有在当前设备已经存在报告快照时显示。展示：

- 成功文件数
- 失败文件数
- 待补处理数
- 最近更新时间
- 校验摘要

并提供跳转：

- 打开任务中心
- 查看导入报告
- 去文件中心
- 打开异常中心

### 6.6 提交区

提交区当前始终存在，支持：

- 保存草稿
- 重新预检
- 提交导入 / 查看任务

提交前会弹出确认对话框。

## 7. 当前导入提交规则

提交按钮可用性来自 `resolveImportSubmitState()`，当前规则如下：

- 没有草稿时不可提交
- 设备已拔出时不可提交
- 会话已在导入中时改为“查看任务”
- 仍有阻塞项时不可提交
- 任一未跳过文件没有目标端时不可提交

## 8. 当前提交后的实际行为

点击确认提交后，当前会：

- 生成一份导入报告快照
- 把设备会话状态改为“导入中”
- 把导入草稿状态改为“导入中”
- 创建一条传输任务
- 为文件清单生成任务子项
- 设置反馈信息
- 切换到任务中心并聚焦该任务

这套流程是当前导入中心最重要的真实闭环。

## 9. 当前弹层与抽屉

当前实际存在的弹层 / 抽屉包括：

- 设备详情抽屉
- 预检详情抽屉
- 导入报告快照抽屉
- 提交导入确认弹层

## 10. 当前数据来源

导入中心当前不直接读 Tauri，也不读 `fileCenterApi`，其数据完全来自：

- `PersistedState.importDeviceSessions`
- `PersistedState.importDrafts`
- `PersistedState.importReports`
- `PersistedState.importSourceNodes`
- `PersistedState.importTargetEndpoints`

这些数据本质上仍是本地 mock 数据。

## 11. 当前与其他页面的联动

### 11.1 与任务中心

- 提交导入后切换任务中心
- 最近结果卡可直接打开任务中心

### 11.2 与异常中心

- 预检摘要卡可打开异常中心
- 结果摘要卡可打开异常中心

### 11.3 与文件中心

- 结果摘要卡可跳到当前资产库的文件中心

### 11.4 与存储节点

- 目标编排卡可查看目标端

## 12. 当前边界与未实现能力

以下能力当前没有实现，文档不应再写成已具备：

- 真实设备发现
- 真实目录扫描
- 真实预检执行器
- 真实导入执行器
- 页内 tab 工作台
- 多会话并排操作
- 真正的导入历史归档页

## 13. 当前版本结论

截至 2026-04-08，导入中心已经形成一套完整的“设备会话式提交前工作台”：

- 左侧设备池，右侧当前会话
- 目标编排、预检、文件级目标勾选、结果摘要全部具备
- 提交导入后能无缝接入任务中心

当前最准确的产品描述应是“本地高保真导入编排工作台”，而不是“真实导入执行系统”。
