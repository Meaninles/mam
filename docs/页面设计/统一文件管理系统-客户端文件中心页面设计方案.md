# 统一文件管理系统-客户端文件中心页面设计方案

## 文档说明

- 更新时间：2026-04-08
- 对照代码：`client/src/pages/FileCenterPage.tsx`、`client/src/pages/FileDetailSheet.tsx`、`client/src/lib/fileCenterApi.ts`、`client/src/App.tsx`
- 目标：用当前实际实现刷新文件中心页面文档

## 1. 页面定位

文件中心是当前客户端的资产主工作台，负责承接以下能力：

- 统一资产浏览
- 目录导航
- 文件搜索与筛选
- 端点同步与删除副本
- 文件标记与标签编辑
- 上传与新建目录
- 进入文件详情

它是当前最核心的一页，也是唯一已经接入独立本地数据库的页面。

## 2. 当前入口与工作区关系

### 2.1 入口

文件中心当前可从以下位置进入：

- 左侧主导航
- 顶层多标签工作区已有标签
- 任务中心详情中的“查看文件中心”
- 异常中心 / 通知中心的回跳定位
- 导入中心中的“去文件中心”

### 2.2 与顶层多标签工作区关系

文件中心运行在顶层多标签工作区中，当前支持：

- 打开后常驻挂载
- 与其他页面并列存在
- 通过标签页右键菜单刷新
- 从任务 / 异常 / 通知跳转时自动切换资产库、目录和选中项

## 3. 当前页面布局

文件中心当前分为四层：

### 3.1 顶部导航工具条

包含：

- 后退
- 前进
- 面包屑地址栏
- 搜索框

### 3.2 操作与筛选工具条

包含：

- 文件类型筛选
- 状态筛选
- 排序方式
- 排序方向切换
- 新建目录
- 上传
- 刷新索引

### 3.3 选中态工具条

仅在有选中项时出现，包含：

- 批量标记
- 批量同步
- 批量删除副本
- 删除资产
- 清空选择

### 3.4 列表与分页区

包含：

- 文件表格
- 空态 / 加载态
- 分页器
- 每页数量切换

## 4. 当前已实现的筛选与搜索

### 4.1 文件类型筛选

当前支持：

- 全部
- 文件夹
- 图片
- 视频
- 音频
- 文档

### 4.2 状态筛选

当前支持：

- 全部
- 完全同步
- 部分同步
- 未同步

其中“部分同步”带二级子菜单，可继续勾选具体端点名。

### 4.3 搜索

当前搜索框提示文案为“搜索名称、路径、来源端”，说明当前搜索范围就是这三类字段。

### 4.4 排序

当前支持：

- 修改时间
- 名称
- 大小
- 星级

并支持升序 / 降序切换。

## 5. 当前列表结构

表格列当前固定为：

- 勾选框
- 名称
- 类型
- 大小
- 修改时间
- 存储状态
- 操作

### 5.1 名称列

名称列会同时展示：

- 文件 / 目录图标
- 文件名
- 标签胶囊
- 星级
- 色标

### 5.2 存储状态列

当前每行会按端点展示一组状态按钮。按钮同时表达：

- 端点类型图标
- 端点名称
- 当前副本状态

如果端点当前允许发起同步，按钮可直接点击。

### 5.3 行操作列

当前行操作包括：

- 打开 / 进入
- 更多操作菜单

更多操作菜单中包含：

- 查看详情
- 标签
- 同步到指定端点
- 删除指定端点副本
- 删除资产

## 6. 当前弹层与抽屉

### 6.1 文件详情抽屉

当前详情抽屉支持展示：

- 文件类型
- 路径
- 来源
- 创建时间
- 最近修改
- 大小
- 最近任务摘要
- 徽标与风险标签
- 元数据列表
- 星级编辑
- 色标编辑
- 备注

### 6.2 标签编辑弹层

当前支持：

- 搜索已有标签
- 勾选 / 取消勾选标签
- 输入并新增新标签
- 查看常用标签使用次数

### 6.3 批量标记弹层

当前仅支持：

- 批量设置星级
- 批量设置色标

当前不支持在这里批量改标签。

### 6.4 新建目录抽屉

当前支持：

- 输入目录名称
- 调用 `fileCenterApi.createFolder`
- 成功后刷新索引

### 6.5 文件动作确认弹层

当前用于以下操作确认：

- 同步确认
- 删除端点副本确认
- 删除资产确认
- 删除冲突确认

当资产仍被运行中的传输任务占用时，弹层会提示冲突，并支持直接跳转任务中心查看阻塞任务。

## 7. 当前已实现的上传能力

### 7.1 上传文件

通过隐藏的文件输入控件实现，可一次选择多个文件。

### 7.2 上传文件夹

通过 `webkitdirectory` / `directory` 属性实现文件夹选择，并把相对路径写入本地数据库。

### 7.3 上传后的当前行为

上传后当前会：

- 在 `fileCenterApi` 中创建目录或文件记录
- 自动补本地端点状态
- 写入基础元数据
- 写入“本地上传”相关徽标

当前不会真正把文件写到操作系统目录，也不会触发真实后台执行器。

## 8. 当前数据来源

### 8.1 页面主数据源

文件中心主数据来自：

- `client/src/lib/fileCenterApi.ts`

### 8.2 存储方式

使用：

- `sql.js`
- `localStorage`

key 为：

- `mare-file-center-sqlite-v4`

### 8.3 与其他页面的边界

当前文件中心的数据和任务 / 异常中心的数据并未完全统一：

- 文件中心自身条目以 `fileCenterApi` 为准
- 任务中心与异常中心的文件引用仍主要依赖 `PersistedState.fileNodes`

因此当前回跳定位是“可用的”，但还不是单一数据库架构。

## 9. 当前与任务中心的联动

### 9.1 发起同步

文件中心发起同步后，会：

- 先调用 `fileCenterApi.syncToEndpoint`
- 再在 `App.tsx` 中创建传输任务
- 任务状态初始为“等待确认”

### 9.2 删除资产

删除资产后，会：

- 调用 `fileCenterApi.deleteAssets`
- 把资产生命周期切到 `PENDING_DELETE`
- 同时创建删除清理类任务

### 9.3 删除端点副本

删除端点副本后，会：

- 调用 `fileCenterApi.deleteFromEndpoint`
- 若无剩余副本，可能直接转为资产删除
- 同时创建删除清理类任务

## 10. 当前已实现边界

以下能力当前尚未实现，文档不应写成已具备：

- 真实文件系统目录映射
- 真实远端副本同步
- 真实后台上传 / 下载执行
- 文件预览器
- 批量标签编辑
- 高级搜索面板
- 用户自定义列

## 11. 当前版本结论

截至 2026-04-08，文件中心已经是一页高完成度的本地资产工作台：

- 有独立数据层
- 有清晰的工具条与表格结构
- 有详情、标签、批量标记、新建目录、上传、同步、删除等闭环
- 能和任务中心、异常中心、通知中心形成真实跳转链路

当前文档应以“本地高保真资产工作台”来描述它，而不是“已接入真实后端的文件管理器”。
