# 统一文件管理系统-客户端文件中心页面设计方案

## 文档信息

- 文档名称：统一文件管理系统-客户端文件中心页面设计方案
- 文档用途：用于客户端“文件中心”页面的产品、交互与界面实现对齐
- 适用范围：Windows 桌面客户端首版
- 当前状态：已按当前客户端实现更新

## 1. 页面定位

当前客户端中的“文件中心”是统一文件管理系统的主工作台。

它不是纯粹的“文件总览”页，也不是独立的“目录视图”页，而是将以下能力合并在一个页面中：

- 资产库内目录式导航
- 文件与目录混合列表浏览
- 存储端状态查看与跨端操作
- 标签、星级、色标等人工标记
- 新建目录、上传、刷新索引等日常动作

页面目标是让用户优先回答“当前目录里有什么、它们同步到了哪里、下一步要怎么处理”，而不是先进入单独的任务工作台或目录树页面。

## 2. 当前实现总览

### 2.1 页面入口

- 文件中心是当前客户端侧边栏的默认主页面，也是顶层多标签工作区默认激活页
- 资产库通过左侧资产库切换器切换
- 导入中心与通知入口位于应用页头右侧

### 2.2 与顶层多标签工作区关系（当前实战）

- 文件中心属于左侧导航一级页面，进入顶层标签体系。
- 当文件中心标签已存在时，再次点击左侧导航“文件中心”会直接聚焦，不会重复打开。
- 文件中心标签关闭通过标签右键菜单完成，不依赖可见 `X` 按钮。
- 文件中心标签可通过右键“向左/向右移动”或直接拖拽整个标签框体调整顺序。
- 在会话内切换到其它顶层页面再返回时，文件中心会保留筛选、搜索、分页、滚动位置、选中项与草稿等上下文。

### 2.3 页面布局

页面当前由 4 层结构组成：

1. 应用页头：资产库切换、导入中心、通知
2. 目录头部：后退、前进、面包屑、搜索框
3. 操作工具条：筛选、排序、新建目录、上传、刷新索引
4. 主体工作区：选择工具条、文件列表、分页器

### 2.4 当前客户端数据来源

当前文件中心是客户端原型实现：

- 数据来源为 `client/src/lib/fileCenterApi.ts`
- 底层使用 `sql.js` 在前端内维护本地 SQLite mock
- 数据会持久化到浏览器 `localStorage`
- 同步状态、删除结果、标签建议等都由客户端 mock 逻辑直接驱动

## 3. 顶部导航与搜索

### 3.1 目录导航

当前目录导航能力包括：

- `后退`
- `前进`
- 面包屑跳转
- 双击目录进入下一级

说明：

- 当前未实现独立目录树
- 目录导航完全依赖面包屑与历史栈

### 3.2 搜索

搜索框位于目录头部右侧。

当前搜索范围为：

- 名称
- 路径
- 来源端

实现说明：

- 输入更新使用前端延迟值，避免高频重载
- 搜索与筛选、排序、分页共同作用于当前资产库 / 当前目录上下文

## 4. 工具条与筛选

### 4.1 文件类型筛选

当前支持：

- `全部`
- `文件夹`
- `图片`
- `视频`
- `音频`
- `文档`

### 4.2 状态筛选

当前支持：

- `全部`
- `完全同步`
- `部分同步`
- `未同步`

补充规则：

- 选择 `部分同步` 时，可继续按端点名称做细分
- 当前端点组合筛选采用 `AND` 关系
- 若某目录的后代文件状态混合，则目录项的端点状态会派生为 `部分同步`

### 4.3 排序

当前支持字段：

- `修改时间`
- `名称`
- `大小`
- `星级`

当前支持：

- 正序
- 逆序

补充规则：

- 目录始终排在文件之前
- 同排序值时，回退到修改时间或名称做稳定排序

### 4.4 操作按钮

当前工具条右侧提供以下动作：

- `新建目录`
- `上传`
- `刷新索引`

其中 `上传` 下拉为：

- `上传文件`
- `上传文件夹`

## 5. 列表区

### 5.1 列表结构

当前主体为表格列表，列定义如下：

- 选择框
- 名称
- 类型
- 大小
- 修改时间
- 存储状态
- 操作

### 5.2 名称列信息

名称列除名称本身外，还会补充：

- 文件类型图标或目录图标
- 标签胶囊（最多显示 3 个）
- 星级
- 色标

### 5.3 大小列规则

- 文件显示格式化后的大小
- 目录显示当前直接子项目数量，例如 `64 项`

### 5.4 存储状态列

存储状态按端点按钮展示，当前端点类型包括：

- 本地
- NAS
- 云端

当前端点状态枚举为：

- `已同步`
- `未同步`
- `同步中`
- `部分同步`

交互规则：

- `未同步`、`部分同步` 可直接点击发起同步
- `已同步` 不提供同步动作
- 目录项的端点状态由后代文件真实状态派生

## 6. 行内操作

### 6.1 快捷动作

每行当前提供两个直接动作：

1. `进入 / 打开详情`
2. `更多操作`

规则：

- 目录点击 `进入`
- 文件点击 `打开详情`

### 6.2 更多操作菜单

当前菜单项包括：

- `查看详情`
- `标签`
- `同步 -> 指定端点`
- `删除 -> 指定端点`
- `删除资产`

补充规则：

- “同步”子菜单只展示当前可同步的端点
- “删除”子菜单只展示当前可删除副本的端点
- 若该目录执行同步或删除副本，动作会递归作用到所有可操作的后代文件

## 7. 批量操作

### 7.1 触发方式

当用户勾选任意文件或目录后，页面会显示选择工具条。

### 7.2 当前批量动作

当前已实现：

- `批量标记`
- `同步 -> 指定端点`
- `删除副本 -> 指定端点`
- `删除资产`
- `清空选择`

### 7.3 确认规则

批量同步与批量删除副本会先弹出确认框。

确认框中会说明：

- 总选择数量
- 已自动跳过的不可操作项数量
- 删除副本后会导致资产被移除的数量

## 8. 详情与弹层

### 8.1 文件详情抽屉

当前文件详情抽屉展示：

- 文件类型
- 路径
- 来源
- 创建时间
- 最近修改
- 大小
- 最近任务
- 徽标 / 风险标签
- 元数据行
- 备注

当前详情抽屉支持编辑：

- 星级
- 色标

### 8.2 标签编辑弹层

标签编辑采用独立弹层。
当前客户端的独立“标签管理”入口已迁移到设置页，本节仅描述文件中心里用于当前资产快速整理的标签编辑弹层。
设置页中的“标签管理”子页面负责全局标签库管理；文件中心弹层只负责当前资产的快速打标。

当前支持：

- 搜索已有标签
- 选择当前资产库可用的常用标签
- 手工输入并新增标签
- 保存到当前资产

实现说明：

- 标签建议当前按资产所在资产库的作用范围加载。
- 若输入的标签不存在，保存时会自动创建到标签库中，并默认归入“未分组”。
- 若标签已存在但当前资产库不在其作用范围内，保存会被拦截并提示先到设置页“标签管理”中调整作用资产库。

### 8.3 批量标记弹层

批量标记当前支持批量更新：

- 星级
- 色标

当前不支持在批量标记中直接批量改标签。

### 8.4 新建目录抽屉

当前新建目录为简单抽屉表单，仅输入：

- 目录名称

校验规则：

- 目录名称不能为空

## 9. 状态与任务语义

### 9.1 同步

当前同步行为在客户端 mock 中表现为：

- 先将目标端点状态设置为 `同步中`
- 同时更新最近任务文案
- 约 5 秒后自动切换为 `已同步`

### 9.2 删除副本

当前从指定端点删除副本时：

- 若仍有其他受管副本，资产继续保留在文件中心
- 若移除了最后一个受管副本，资产会被直接从文件中心移除

### 9.3 删除资产

当前删除资产会将条目标记为：

- 生命周期 `PENDING_DELETE`
- 各端点状态更新为 `未同步`

用户侧反馈为“删除请求已提交，资产进入等待清理”。
后续删除执行与后台清理会在“任务中心 > 其它任务”中以“删除清理任务”继续跟踪。
该页面当前在 `运行中` 与 `等待清理` 状态下都支持暂停，暂停后可在同页继续。

## 10. 当前边界

以下能力当前未在文件中心实现：

- 独立目录树
- 缩略图视图
- 重命名
- 移动
- 复制
- 详情内直接发起移动 / 复制 / 打开所在目录
- 常用筛选保存
- 按扩展名、拍摄时间、设备、分辨率、时长等高级筛选

## 11. 当前实现结论

当前客户端文件中心已经明确为：

- 以目录式导航承接用户文件夹使用习惯
- 以统一资产列表承接多端状态查看
- 以端点按钮、更多菜单和批量工具条承接跨端动作
- 以详情抽屉和标签 / 标记弹层承接轻量资产整理
- 以本地 SQLite mock 验证首版交互闭环
- 文件中心继续作为重命名、路径调整、目录核对等强业务动作的原场景，不取代异常中心的统一治理职责
- 异常中心当前已可按 `libraryId / fileNodeId / path` 回跳文件中心查看上下文对象，后续接入后端时不应改变这一交互分工

后续文档与开发若继续围绕文件中心迭代，应以本版本结构与交互为准。
