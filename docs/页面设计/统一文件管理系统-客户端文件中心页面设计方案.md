# 统一文件管理系统-客户端文件中心页面设计方案

## 文档说明

- 更新时间：2026-04-11
- 对照代码：`client/src/pages/FileCenterPage.tsx`、`client/src/pages/FileDetailSheet.tsx`、`client/src/lib/fileCenterApi.ts`、`client/src/App.tsx`
- 目标：用当前实际实现刷新文件中心页面文档

## 1. 页面定位

文件中心是当前客户端的资产主工作台，负责承接以下能力：

- 统一资产浏览
- 目录导航
- 文件搜索与筛选
- 端点同步与删除副本
- 文件标记与标签编辑
- 上传与新建目录
- 进入文件详情

它是当前最核心的一页，现已形成“中心服务主链路 + 本地补充能力”的混合实现：资产、目录、副本状态优先来自 center 服务，标签管理与部分端点动作仍保留本地实现。

## 2. 当前入口与工作区关系

### 2.1 入口

文件中心当前可从以下位置进入：

- 左侧主导航
- 顶层多标签工作区已有标签
- 任务中心详情中的“查看文件中心”
- 异常中心 / 通知中心的回跳定位
- 导入中心中的“去文件中心”

### 2.2 与顶层多标签工作区关系

文件中心运行在顶层多标签工作区中，当前支持：

- 打开后常驻挂载
- 与其他页面并列存在
- 通过标签页右键菜单刷新
- 从任务 / 异常 / 通知跳转时自动切换资产库、目录和选中项

## 3. 当前页面布局

文件中心当前分为四层：

### 3.1 顶部导航工具条

包含：

- 后退
- 前进
- 面包屑地址栏
- 搜索框

### 3.2 操作与筛选工具条

包含：

- 文件类型筛选
- 状态筛选
- 排序方式
- 排序方向切换
- 新建目录
- 上传
- 刷新索引

### 3.3 选中态工具条

仅在有选中项时出现，包含：

- 批量标记
- 批量同步
- 批量删除副本
- 删除资产
- 清空选择

### 3.4 列表与分页区

包含：

- 文件表格
- 空态 / 加载态
- 分页器
- 每页数量切换

## 4. 当前已实现的筛选与搜索

### 4.1 文件类型筛选

当前支持：

- 全部
- 文件夹
- 图片
- 视频
- 音频
- 文档

### 4.2 状态筛选

当前支持：

- 全部
- 完全同步
- 部分同步
- 未同步

其中“部分同步”带二级子菜单，可继续勾选具体端点名。

### 4.3 搜索

当前搜索框提示文案为“搜索名称、路径、来源端”，说明当前搜索范围就是这三类字段。

### 4.4 排序

当前支持：

- 修改时间
- 名称
- 大小
- 星级

并支持升序 / 降序切换。

## 5. 当前列表结构

表格列当前固定为：

- 勾选框
- 名称
- 类型
- 大小
- 修改时间
- 存储状态
- 操作

### 5.1 名称列

名称列会同时展示：

- 文件 / 目录图标
- 文件名
- 标签胶囊
- 星级
- 色标

### 5.2 存储状态列

当前每行会按端点展示一组状态按钮。按钮同时表达：

- 端点类型图标
- 端点名称
- 当前副本状态

如果端点当前允许发起同步，按钮可直接点击。

### 5.3 行操作列

当前行操作包括：

- 打开 / 进入
- 更多操作菜单

更多操作菜单中包含：

- 查看详情
- 标签
- 同步到指定端点
- 删除指定端点副本
- 删除资产

## 6. 当前弹层与抽屉

### 6.1 文件详情抽屉

当前详情抽屉支持展示：

- 文件类型
- 路径
- 来源
- 创建时间
- 最近修改
- 大小
- 最近任务摘要
- 徽标与风险标签
- 星级编辑
- 色标编辑

### 6.2 标签编辑弹层

当前支持：

- 搜索已有标签
- 勾选 / 取消勾选标签
- 输入并新增新标签
- 查看常用标签使用次数

### 6.3 批量标记弹层

当前仅支持：

- 批量设置星级
- 批量设置色标

当前不支持在这里批量改标签。

### 6.4 批量标签弹层

当前批量标签弹层支持：

- 以选中条目的标签并集作为可操作集合，而不是只显示共同标签
- 勾选标签时，为缺失该标签的条目补标签，已拥有该标签的条目自动跳过
- 取消标签时，从拥有该标签的条目中移除，本来没有该标签的条目自动跳过
- 保留同条目中未被取消的其它标签，不会因为批量取消一个标签而误删另一个标签

### 6.5 新建目录抽屉

当前支持：

- 输入目录名称
- 调用 `fileCenterApi.createFolder`
- 成功后刷新索引

### 6.6 文件动作确认弹层

当前用于以下操作确认：

- 同步确认
- 删除端点副本确认
- 删除资产确认
- 删除冲突确认

当资产仍被运行中的传输任务占用时，弹层会提示冲突，并支持直接跳转任务中心查看阻塞任务。

## 7. 当前已实现的上传能力

### 7.1 上传文件

通过隐藏的文件输入控件实现，可一次选择多个文件。

### 7.2 上传文件夹

通过 `webkitdirectory` / `directory` 属性实现文件夹选择，并把相对路径随上传清单一并提交给中心服务。

### 7.3 上传后的当前行为

上传后当前会：

- 在非 `test` 环境下调用 `POST /api/libraries/{libraryId}/uploads`
- 将文件内容真实写入 LOCAL / NAS 的可写挂载目录
- 同步回写 `assets`、`asset_replicas`、`directory_presences` 等索引数据
- 刷新后可在文件中心继续浏览刚写入的目录和文件

当前仍未具备：

- CLOUD 端点的真实上传执行
- 后台作业化上传、进度回写、失败重试
- 标签随上传一起写入后端

## 8. 当前数据来源

### 8.1 页面主数据源

文件中心主数据入口来自：

- `client/src/lib/fileCenterApi.ts`

### 8.2 存储方式

当前分成两类：

- 中心服务主链路：
  - `GET /api/libraries`
  - `GET /api/libraries/{libraryId}/browse`
  - `GET /api/file-entries/{id}`
  - `POST /api/libraries/{libraryId}/directories`
  - `POST /api/libraries/{libraryId}/scan`
  - `POST /api/libraries/{libraryId}/uploads`
  - `GET /api/tags/suggestions`
  - `PATCH /api/file-entries/{id}/annotations`
  - `DELETE /api/file-entries/{id}`
- 本地保留链路：
  - `sql.js` + `localStorage`
  - key：`mare-file-center-sqlite-v4`
  - 当前主要承载 `test` 运行时的文件中心 fallback，以及少量测试辅助镜像；切片 7 对应的端点重操作已不再依赖这里的原型态实现

### 8.3 与其他页面的边界

当前文件中心的数据和任务 / 异常中心的数据并未完全统一：

- 文件中心的资产、目录、详情、扫描索引、星级 / 色标、标签、上传、新建目录、资产删除优先以中心服务返回结果为准
- 其中目录扫描已经切到真实作业链路：调用 `/api/libraries/{libraryId}/scan` 会创建真实扫描作业并返回 `jobId`
- 标签、标签建议、文件/目录标签保存已走中心服务与 PostgreSQL；其中批量标签采用按并集增量应用的真实语义。端点同步、删除端点副本、删除资产现在都会通过中心服务创建真实作业
- 任务中心已经改为消费中心服务真实作业数据，文件中心扫描结果会在任务中心“其他任务”中可见；异常中心的文件引用仍主要依赖 `PersistedState.fileNodes`

因此当前回跳定位是“可用的”，但还不是单一数据库架构。

## 9. 当前与任务中心的联动

### 9.1 发起同步

文件中心发起同步后，会：

- 调用 `POST /api/file-entries/replicate`
- 由中心服务创建 `REPLICATE` 真实作业
- 任务中心通过既有 `jobsApi` + `/api/events/stream` 消费该作业
- 文件中心在目标端点存在运行中同步任务时可显示“同步中”

当前真实执行仅支持 `LOCAL` / `NAS`；`CLOUD` 端点仅保留正式接口与计划编排入口，后续接入 CD2 / aria2 后再落地执行。

### 9.2 删除资产

删除资产后，会：

- 调用 `POST /api/file-entries/delete-assets`
- 由中心服务创建 `DELETE_ASSET` 真实作业
- 作业先删除受支持端点上的资产副本，再按目录深度倒序删除被选中的空目录
- 任务中心展示真实删除作业，文件中心根据真实作业状态刷新最近任务与副本状态

### 9.3 删除端点副本

删除端点副本后，会：

- 调用 `POST /api/file-entries/delete-replicas`
- 由中心服务创建 `DELETE_REPLICA` 真实作业
- 删除完成后，若某资产已无剩余副本，资产会被真实清理
- 文件中心端点状态与最近任务文案会按真实作业结果回写

## 10. 当前已实现边界

以下能力当前尚未实现，文档不应写成已具备：

- `CLOUD` 端点的真实上传 / 下载 / 删副本执行（后续由 CD2 / aria2 接入）
- 超出当前切片范围的云端断点续传、网盘链路专用重试治理
- 文件预览器
- 文件中心按标签查询/筛选（后端 `browse` 仍未提供标签筛选参数）
- 高级搜索面板
- 用户自定义列

## 11. 当前版本结论

截至 2026-04-12，文件中心已经是一页“中心服务主链路为主、测试环境保留少量本地 fallback”的资产工作台：

- 资产、目录、详情、扫描、新建目录、上传、星级 / 色标、标签、同步、删除副本、删除资产都已接入真实 center API / 真实作业体系
- 标签建议与标签保存已后端化；切片 7 对应的端点重操作已完成服务化
- 有清晰的工具条与表格结构
- 有详情、标签、批量标记、新建目录、上传、同步、删除等闭环
- 能和任务中心、异常中心、通知中心形成真实跳转链路

当前文档应把它描述为“文件中心主读取、轻写入与切片 7 的重操作都已进入后端权威链路；仅 `test` fallback 与后续云端执行链路仍保留边界”的现实实现，而不能继续写成“端点重操作仍未服务化”。

补充口径：

- 文件中心“扫描刷新索引”已不再是同步直跑，而是切入真实作业内核
- 这批真实扫描作业现在已经会被任务中心“其他任务”页消费并展示
- 文件中心真实同步链路会保留源文件最后修改时间（`mtime`），用于维持文件原始排序稳定性
