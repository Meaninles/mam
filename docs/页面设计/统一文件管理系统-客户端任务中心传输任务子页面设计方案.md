## 1. 页面定位

“任务中心 > 传输任务”是统一文件管理系统客户端中的传输任务监控与管理工作台。

它不是新的发起入口页，也不是异常处理页，而是用于统一承接和管理以下已经发起的传输任务：

- 导入任务 `IMPORT`
- 同步任务 `SYNC`

其中：

- 页面顶层只保留 `导入` 和 `同步` 两类业务任务
- `同步` 在展示层继续细分为 `复制`、`上传`、`下载`
- `复制 / 上传 / 下载` 是同步链路标签，不是一级业务类型
- 页面不直接处理异常，只提供异常提示、异常浮窗和跳转异常中心能力

页面目标是让用户优先回答以下问题：

- 现在有哪些传输任务正在跑
- 哪些任务暂停了、失败了、需要我处理
- 某个任务当前传到哪一步了
- 这个任务里包含哪些文件或文件夹
- 这个任务触发了哪些异常，应该去哪里处置

### 1.1 与顶层多标签工作区关系（当前实战）

- 任务中心属于左侧导航一级页面，进入顶层标签体系。
- “任务中心”标签已存在时，再次点击左侧导航会直接聚焦，不会重复打开。
- 顶层标签关闭通过右键菜单执行，不使用可见 `X` 按钮。
- 标签排序支持右键“向左/向右移动”与整块标签框体拖拽。
- 会话内在文件中心、异常中心、存储节点、设置之间切换后，返回任务中心可保留任务页筛选、搜索、排序与子页状态。

## 2. 设计原则

### 2.1 生产可用优先

本方案以“先把 UI/UX 定型，后续接后端时尽量不改界面”为目标。

因此本页面的结构、字段、状态表达、交互入口和异常联动要尽量一次定清，不依赖后端实现细节反复调整界面。

### 2.2 列表优先

该页面的主要内容必须是传输任务列表。

因此以下内容都应从属于列表，不应喧宾夺主：

- 顶部概览统计
- 高级筛选
- 批量操作条
- 详情抽屉
- 异常浮窗

### 2.3 与现有客户端风格一致

本页面视觉与交互必须延续当前客户端已有设计语言，重点参考：

- 文件中心页面：工具条、搜索、筛选、列表密度、菜单与抽屉风格
- 存储节点页面：子页切换、轻量页头、紧凑列表、上下文菜单、反馈方式

明确要求：

- 继续使用当前客户端统一的卡片、边框、圆角、按钮、胶囊、抽屉、工具条体系
- 不引入与当前客户端风格割裂的“大屏监控面板”或“重图表看板”
- 不使用明显不同的布局体系或第三方 UI 风格

### 2.4 任务级管理优先

首版以“任务级管理”为主，不做子项级批量管理。

也就是说：

- 支持任务级批量暂停、继续、取消、重试、修改优先级
- 不支持对二级子项或三级文件节点做批量控制

## 3. 业务范围与边界

### 3.1 纳入范围

本页面纳入以下业务任务：

- `IMPORT`
- `SYNC`

其中 `SYNC` 在界面上继续展示链路类型：

- `复制`
  - 本地到本地
  - NAS 到 NAS
  - 本地与 NAS 之间
- `上传`
  - 本地 / NAS 到网盘
- `下载`
  - 网盘到本地 / NAS

### 3.2 不纳入范围

以下能力不在本页面直接完成：

- 在页内处置异常
- 在页内解决冲突
- 在页内编辑源路径或目标路径
- 在页内发起全新的导入批次
- 在页内做子项级批量操作

### 3.3 与其它页面的关系

- 文件中心：传输任务的主要发起页之一
- 导入中心：导入批次与导入任务的前置发起页
- 异常中心：异常的统一处置页

传输任务页只负责：

- 看
- 管
- 跳

不负责：

- 发起复杂配置
- 处理异常闭环

### 3.4 后续 NAS 部署演进补充

- 当前阶段传输任务仍按 Windows 本机部署口径运行。
- 后续若采用 `Central Core + Local Edge Agent` 架构，本页看到的所有任务、进度、异常与结果仍统一来自 `Central Core`。
- 当链路涉及 Windows 本地磁盘、移动硬盘、U 盘等本地读写动作时，可由 `Local Edge Agent` 执行实际传输并回传进度。
- 本页不直接展示 `Local Edge Agent` 的原生任务视角，而是继续展示 `Central Core` 汇总后的统一传输任务模型。

## 4. 页面结构

页面建议沿用当前客户端的轻量四层结构。

### 4.1 页头层

页头保持与当前客户端一致，不在本子页面重新定义应用级头部。

子页面内部只承接已有页头下的任务工作区。

### 4.2 子页切换层

“任务中心”下保留现有子页切换结构，但“传输任务”子页面内部不再使用现在的“传输任务 / 其它任务”简陋切换。

本子页面内采用两段式业务筛选：

1. 一级业务切换
   - `导入`
   - `同步`
2. 二级链路筛选
   - `全部`
   - `复制`
   - `上传`
   - `下载`

说明：

- 当一级切到 `导入` 时，二级链路筛选自动隐藏或禁用
- 当一级切到 `同步` 时，二级链路筛选显示

### 4.3 工具条层

工具条保持紧凑，参考文件中心和存储节点页面。

建议从左到右依次为：

1. 一级业务切换
2. 二级链路筛选
3. 状态筛选
4. 资产库筛选
5. 搜索框
6. 排序方式
7. 批量操作入口

说明：

- 工具条要轻，不要额外叠大块统计卡
- 批量操作只在有选择项时显著显示

### 4.4 列表主体层

页面主体必须优先展示传输任务列表。

当前实现里默认只展示一级任务，但一级摘要和展开内容必须按同一递归文件语义理解。

用户手动展开后，可在列表中查看二级任务。

对于文件夹型一级任务，展开后只显示递归提取出的文件，不单独渲染文件夹节点。

三级任务不在列表中直接展开，统一在详情抽屉中查看。

### 4.5 详情抽屉层

详情抽屉为右侧抽屉，延续当前 `Sheet` 风格，但信息密度高于现有任务详情原型。

抽屉用于查看：

- 任务概览
- 二级与三级任务层级
- 子项状态
- 异常摘要
- 跳转入口

## 5. 页面布局与视觉风格

### 5.1 整体布局

建议采用与文件中心一致的“工具条卡片 + 紧凑列表卡片”结构：

1. 工具条卡片
2. 列表卡片
3. 详情抽屉

不新增复杂分栏，不做独立左树，不做右侧常驻详情栏。

### 5.2 信息密度

列表行密度应参考当前文件中心和存储节点页面，偏紧凑但可读。

原则：

- 主信息一眼可读
- 次级信息用副标题和小胶囊表达
- 非核心信息不新增列

### 5.3 视觉重点

页面视觉焦点顺序必须是：

1. 任务名称与状态
2. 进度与传输速度
3. 异常提示
4. 可操作按钮

不应把以下内容做成视觉主角：

- 统计卡
- 批量条
- 高级过滤器

## 6. 列表设计

### 6.1 列表展示规则

当前实现里，默认列表仅显示一级任务。

用户点击展开按钮后，在当前任务行下方展开二级任务区域。

二级任务用于承接递归后的文件明细，当前实现中不再单独把文件夹节点作为二级任务渲染：

- 文件项
- 分发目标节点

三级任务统一只在详情抽屉查看，不在列表里继续无限展开。

### 6.2 一级任务行字段

一级任务行建议固定包含以下字段：

- 选择框
- 展开按钮
- 任务名称
- 唯一路径小字（仅在存在唯一路径时显示）
- 主业务标签：`导入` / `同步`
- 次级链路标签：`复制` / `上传` / `下载`
- 进度条下方统一显示源端点 -> 目标端点
- 大小胶囊：有二级任务时显示 `xx项`，否则显示大小
- 总进度
- 速度
- 剩余时间
- 状态胶囊
- 优先级
- 异常提示图标
- 操作区

说明：

- 不显示“最近更新时间”
- 不显示底层执行器名称
- 不显示外部任务 ID
- 一级摘要口径与展开内容保持一致，均按同一递归文件语义计算

### 6.3 二级任务展开区字段

二级任务展开区建议采用轻量内嵌行列表，不再显示完整主任务字段。

建议字段为：

- 名称
- 名称下方路径
- 节点类型：子任务 / 文件 / 目标分发项
- 状态
- 当前阶段
- 进度
- 大小或文件数摘要
- 异常提示图标

当前实现里，二级任务名称下方显示路径；若关联文件已删除，则整行进入红灰失效态，文本带删除线，且行内所有控件不可点击。

二级任务区不放完整操作条，只保留与一级一致的状态按钮规则和少量查看能力。

当前实现中，暂停 / 继续 / 重试互斥渲染，已完成状态保留 `完成` 按钮占位。

### 6.4 异常提示图标

异常数不以普通文本列展示，而是以“小红色数字图标”展示。

建议规则：

- 无异常：不显示图标
- 有异常：显示红色异常徽标 + 数字
- 数字上限显示 `99+`

交互：

- 点击图标，弹出异常浮窗
- 浮窗锚定在该任务行的异常图标位置

## 7. 异常浮窗设计

### 7.1 浮窗定位

异常浮窗为轻量浮层，不进入详情抽屉，也不占据列表主要空间。

浮窗应：

- 锚定异常图标
- 点击外部关闭
- 支持滚动查看异常列表

### 7.2 浮窗内容

浮窗内展示当前任务关联的异常摘要列表，建议字段为：

- 异常严重级别
- 异常类型
- 简短说明
- 所属子项名称
- 当前处置状态

### 7.3 浮窗操作

浮窗底部固定两个动作：

1. `处置所有异常`
   - 跳转异常中心
   - 自动带上该任务的异常筛选条件
2. 单条异常点击
   - 跳转异常中心
   - 自动定位到该异常记录

说明：

- 异常中心的具体筛选实现后续再做
- 本页面先把交互入口与参数契约定型

### 7.4 当前实现校验点（2026-04-07）

- 当前客户端已按本节方向落地异常浮窗，浮窗锚定在任务行异常徽标附近，点击空白区域关闭。
- 点击浮窗中的单条异常，会跳转异常中心并按 `issueId` 聚焦到对应异常记录。
- 点击 `处置所有异常`，会跳转异常中心并带上 `taskId + sourceDomain` 作为聚焦条件。
- 从异常中心反向打开任务中心时，当前客户端会自动选中对应任务，并可直接展开异常浮窗回查原任务上下文。

## 8. 详情抽屉设计

### 8.1 抽屉定位

详情抽屉是“深度查看”容器，不是主要工作区。

列表仍然是主舞台，抽屉只在需要深查时打开。

### 8.2 抽屉结构

建议分为四个区块：

#### 1. 任务概览

- 任务名称
- 业务类型
- 同步链路标签
- 当前状态
- 优先级
- 发起来源
- 源端点
- 目标端点
- 源路径
- 目标路径
- 总体进度
- 速度
- 剩余时间
- 文件数
- 文件夹数

#### 2. 层级结构

以树状结构展示：

- 一级：主任务
- 二级：子任务 / 文件夹 / 分发节点
- 三级：文件

说明：

- 三级任务只在抽屉内展示
- 抽屉内允许完整展开

#### 3. 子项明细

用于补充显示当前树节点的细节：

- 状态
- 当前阶段
- 进度
- 大小
- 速度
- 校验状态

#### 4. 异常摘要

- 当前任务异常总数
- 最近异常
- 高风险异常数
- 跳转异常中心
- 跳转文件中心

### 8.3 抽屉操作

抽屉顶部或底部操作区建议保留：

- `暂停`
- `继续`
- `取消`
- `重试`
- `修改优先级`
- `查看文件中心`
- `查看异常中心`

说明：

- 动作按钮按任务能力动态显示
- 不展示当前状态不支持的操作

## 9. 筛选、排序与搜索

### 9.1 一级业务筛选

固定为：

- `导入`
- `同步`

### 9.2 二级链路筛选

仅在一级切到 `同步` 时显示：

- `全部`
- `复制`
- `上传`
- `下载`

### 9.3 状态筛选

当前实现里，状态筛选新增 `活跃中`，且默认选中 `活跃中`，它聚合 `运行中` / `已暂停` / `等待确认` 三类状态。

建议支持的筛选顺序为：

- `全部`
- `活跃中`
- `运行中`
- `已暂停`
- `等待确认`
- `异常待处理`
- `部分成功`
- `失败`
- `已完成`
- `已取消`

### 9.4 资产库筛选

支持按资产库过滤任务。

### 9.5 搜索

搜索建议覆盖：

- 任务名称
- 源路径
- 目标路径
- 文件名
- 文件夹名
- 异常摘要

### 9.6 排序

页面不展示“最近更新时间”，但内部仍需有稳定排序依据。

推荐默认排序规则：

1. `运行中`
2. `异常待处理`
3. `已暂停`
4. `等待确认`
5. 其它状态

同状态下再按：

1. 优先级
2. 创建时间倒序

界面可提供的排序方式建议为：

- `默认排序`
- `优先级`
- `进度`
- `名称`

## 10. 状态表达

### 10.1 主任务状态

主任务建议统一使用以下状态语义：

- `待调度`
- `运行中`
- `等待确认`
- `已暂停`
- `异常待处理`
- `部分成功`
- `已完成`
- `失败`
- `已取消`

其中，`活跃中` 是筛选层的聚合状态，不是任务底层状态。

### 10.2 子项状态

子项和树节点建议使用更细粒度状态：

- `待预检`
- `可执行`
- `传输中`
- `校验中`
- `提交中`
- `重试等待`
- `已阻塞`
- `已暂停`
- `已完成`
- `失败`
- `已取消`

### 10.3 显示规则

- 页面展示的是系统统一任务状态，不直接暴露执行器原始状态
- 即使外部执行器显示完成，只有本地任务状态进入完成语义后才展示为 `已完成`
- `校验中` 不得显示为完成
- `部分成功` 要明确显示，不可被归并成完成
- 当前实现里，一级任务状态背景使用淡色强调：`运行中` 为淡黄，`失败` 为淡红，`已完成` 为淡绿

## 11. 操作设计

### 11.1 单任务操作

首版支持：

- 暂停
- 继续
- 取消
- 重试
- 修改优先级
- 打开详情

当前实现里，一级和二级任务都按同一套互斥规则渲染按钮：`暂停` / `继续` / `重试` 互斥，`已完成` 保留 `完成` 按钮占位，`取消` 始终保留。

### 11.2 批量操作

首版支持的批量操作：

- 批量暂停
- 批量继续
- 批量取消
- 批量重试
- 批量修改优先级
- 清空选择

### 11.3 跳转操作

首版支持以下跳转：

- 跳转异常中心
- 跳转文件中心

说明：

- 跳转文件中心优先定位到任务关联资产或路径
- 跳转异常中心优先带 `taskId`

### 11.4 能力型按钮规则

按钮不应只按状态硬编码判断，而应按“能力模型”控制。

例如：

- 当前任务可暂停时才显示 `暂停`
- 当前任务可继续时才显示 `继续`
- 当前任务可重试时才显示 `重试`

## 12. 空态、加载态与错误态

### 12.1 无任务空态

当没有任何传输任务时，空态文案应提示用户：

- 可去导入中心提交导入任务
- 可在文件中心发起同步任务

### 12.2 筛选结果为空

当只是当前筛选条件下无结果时，应提示：

- 当前筛选下没有匹配任务
- 提供一键清空筛选

### 12.3 加载态

首屏进入和切换筛选时，建议使用轻量骨架屏，不使用大面积加载转圈。

### 12.4 错误态

任务列表加载失败时：

- 在列表卡片内显示错误提示
- 提供 `重试` 按钮

## 13. 一致性要求

### 13.1 与文件中心的一致性

应保持以下一致：

- 工具条卡片结构
- 搜索框形式
- 筛选器样式
- 列表卡片密度
- 行内按钮与更多操作风格
- 详情抽屉风格

### 13.2 与存储节点的一致性

应保持以下一致：

- 子页切换方式
- 紧凑页内布局
- 轻量反馈方式
- 低频动作进入菜单

### 13.3 明确不建议的设计

以下设计不应采用：

- 单独做成监控大屏
- 大量统计图表占据上半屏
- 常驻右栏详情
- 把异常处理流程直接塞进当前页
- 无限层级在列表内展开

## 14. 技术架构设计

## 14.1 设计目标

技术方案要保证：

- 先做前端 UI，后接后端时不需要改动主要 UI/UX
- 列表、抽屉、异常浮窗可用统一视图模型驱动
- 不耦合具体执行器实现
- 支持后续接入真实轮询、推送、任务恢复和异常中心筛选
- 后续若核心迁移到 NAS，仍由 `Central Core` 作为唯一权威状态源，`Local Edge Agent` 只作为本地链路执行与事实回传方

## 14.3 查询与命令分离

建议前端与后端边界分为两类：

### 查询接口

用于读取：

### 命令接口

用于执行：

- 暂停
- 继续
- 取消
- 重试
- 调整优先级
- 批量暂停
- 批量继续
- 批量取消
- 批量重试
- 批量调整优先级

说明：

- UI 不关心底层是 Native、CloudDrive2 还是 aria2
- UI 只消费统一查询结果和统一能力字段

## 14.4 异常关联模型

为了保证后续异常中心无需反推 UI，异常关联建议统一稳定键：

- `taskId`
- `taskItemId`
- `issueId`
- `libraryId`
- `endpointId`
- `assetId`

本页面只使用其中一部分，但这些键应提前定型。

用途：

- 当前页按任务展示异常数量和异常浮窗
- 后续异常中心可按任务筛出全部异常
- 文件中心可按资产反查最近异常

## 14.5 选择模型与批量操作

批量操作只做任务级选择。

建议前端维护：

- `selectedTaskIds`
- `allVisibleSelected`
- `batchCapabilities`

批量能力计算规则：

- 按选中任务能力交集计算
- 只显示所有选中任务都支持的动作

这样可避免：

- 展示无效批量按钮
- 用户选中混合状态任务后不知道为什么操作失败

## 14.6 列表与详情的数据装载策略

为保证后续生产环境性能，建议数据加载分层：

### 首屏列表

只加载一级任务摘要。

### 二级任务展开

用户展开某个一级任务时，再按需加载该任务二级节点。

### 详情抽屉

用户打开详情抽屉时，再加载：

- 完整任务详情
- 完整节点树
- 三级节点
- 异常摘要列表

这样可以保证：

- 首屏快
- 列表轻
- 深查时信息仍然完整

## 14.7 刷新与实时同步策略

建议后续接入时采用分层刷新：

- 运行中任务：高频轮询或推送
- 已暂停 / 等待确认：中频刷新
- 已完成 / 已取消 / 失败：低频刷新或不自动刷新

说明：

- 页面显示状态应始终以本地统一任务状态为准
- 外部执行器状态只作为内部事实输入，不直接映射为 UI 终态

## 16. 必须定稿的约束

以下内容在后续实现前应视为定稿，不建议反复修改：

- 页面顶层只保留 `导入 / 同步`
- `同步` 的二级链路标签为 `复制 / 上传 / 下载`
- 默认只展示一级任务
- 二级任务允许在列表中手动展开
- 三级任务只在详情抽屉查看
- 列表不展示最近更新时间
- 异常提示使用红色小数字图标
- 异常图标点击后弹出异常浮窗
- 异常浮窗支持“处置所有异常”
- 页面只做异常提示与跳转，不做异常处置
- 本页面设计风格必须与文件中心、存储节点保持一致

## 17. 结论

本方案将“任务中心 > 传输任务”明确为一个以列表为核心的生产可用传输任务监控台。

它在业务上统一收敛导入与同步任务，在界面上统一承载复制、上传、下载三类链路，在交互上把异常处理入口前移但不把异常处理过程塞回本页，在技术上通过稳定的视图模型、能力模型和异常关联模型，为后续后端接入保留足够空间。

按本方案实现后，后续在接入真实后端、CloudDrive2、aria2、异常中心筛选和文件中心联动时，应尽量只替换数据来源与命令实现，而不再改动本子页面的主要 UI、UX 和信息架构。
