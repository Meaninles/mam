## 1. 阶段1：后台调度器与定时扫描

### 1.1 定时扫描任务自动创建

- 状态：已通过
- 前置条件：
  - 中心服务已启动
  - 至少存在一个挂载，且扫描策略为“定时扫描”
  - PostgreSQL 正常可用
- 操作步骤：
  1. 打开客户端，进入“存储节点 -> 挂载文件夹”。
  2. 新建或编辑一个挂载，设置扫描策略为“定时扫描”。
  3. 关闭客户端，保持中心服务继续运行。
  4. 等待后台调度周期触发。
  5. 重新打开客户端，进入“任务中心 -> 其他任务/维护任务”。
- 预期结果：
  - 可以看到后台自动创建的扫描任务。
  - 任务来源不依赖当前客户端是否在线。
  - 扫描任务完成后，挂载扫描历史可继续查询。
- 通过依据：
  - `services/center/internal/governance` 集成测试通过
  - `services/center/internal/storage` 集成测试通过

## 2. 阶段2：心跳与健康检查闭环

### 2.1 到期心跳自动创建连接测试作业

- 状态：已通过
- 前置条件：
  - 已存在挂载，心跳策略不为“从不”
  - 中心服务持续运行
- 操作步骤：
  1. 打开客户端，进入“存储节点 -> 挂载文件夹”。
  2. 将某个挂载的心跳策略设置为“每小时”或“每日（深夜）”。
  3. 关闭客户端。
  4. 等待后台到期调度。
  5. 重新打开客户端，进入“任务中心”与“异常中心”。
- 预期结果：
  - 后台会自动创建连接测试作业。
  - 若目标不可用，作业失败后会沉淀为异常，并进入通知中心。
  - 若目标可用，运行态与挂载状态会被刷新。
- 通过依据：
  - `services/center/internal/governance` 心跳调度测试通过
  - `services/center/internal/issues`、`services/center/internal/notifications` 集成测试通过

## 3. 阶段3：自动重试策略

### 3.1 可恢复失败自动退避重试

- 状态：已通过
- 前置条件：
  - 存在会触发可恢复失败的连接测试或传输任务
  - 中心服务持续运行
- 操作步骤：
  1. 打开客户端，触发一个连接测试或后台巡检任务。
  2. 让目标在第一次执行时失败，例如临时断开目标路径或故意制造短暂不可达。
  3. 观察任务中心中的状态变化。
  4. 在退避时间结束后恢复目标可用性。
- 预期结果：
  - 任务先进入“等待重试”。
  - 到达退避时间后，系统自动再次执行。
  - 恢复目标可用后，任务最终完成，不需要人工手动点“重试”。
- 通过依据：
  - `services/center/internal/jobs` 自动重试集成测试通过

### 3.2 超过预算后进入失败与治理

- 状态：已通过
- 前置条件：
  - 可恢复失败目标持续保持失败状态
- 操作步骤：
  1. 触发一个后台连接测试或治理任务。
  2. 保持失败条件不恢复。
  3. 观察任务中心、异常中心和通知中心。
- 预期结果：
  - 系统会按预算自动重试。
  - 达到重试上限后，任务转为失败。
  - 失败结果进入异常中心，可继续治理。
- 通过依据：
  - `services/center/internal/jobs` 自动重试预算集成测试通过

## 4. 阶段4：文件消失检测治理

### 4.1 缺失副本转为治理异常与通知

- 状态：已通过
- 前置条件：
  - 文件已完成一次扫描建档
  - 物理文件被外部删除或移动
- 操作步骤：
  1. 打开客户端，确认文件已出现在文件中心。
  2. 在操作系统中直接删除或移走对应物理文件。
  3. 等待后台巡检或再次触发扫描。
  4. 打开“异常中心”和“通知中心”。
- 预期结果：
  - 缺失副本不会被静默忽略。
  - 系统会生成治理异常，来源域为“系统治理”。
  - 通知中心会出现对应提醒，并可跳转定位。
- 通过依据：
  - `services/center/internal/issues` 缺失副本治理测试通过
  - `services/center/internal/governance` 缺失治理调度测试通过

## 5. 阶段5：深度校验

### 5.1 副本校验成功回写校验状态

- 状态：已通过
- 前置条件：
  - LOCAL 或 NAS 副本存在且可读
- 操作步骤：
  1. 打开客户端，确保文件已经扫描入库。
  2. 触发副本校验任务。
  3. 等待任务完成。
  4. 查看任务中心与副本状态。
- 预期结果：
  - 校验任务成功完成。
  - 副本 `verification_state` 更新为 `PASSED`。
  - 快速哈希与校验时间写回数据库。
- 通过依据：
  - `services/center/internal/assets` 副本校验集成测试通过

### 5.2 校验发现内容不一致后进入 VERIFY 异常

- 状态：已通过
- 前置条件：
  - 副本已有一次成功校验
  - 物理文件内容被修改
- 操作步骤：
  1. 先完成一次成功校验。
  2. 在操作系统中修改同一路径文件内容。
  3. 再次触发副本校验任务。
  4. 打开“任务中心”和“异常中心”。
- 预期结果：
  - 校验任务失败。
  - 异常中心出现 `VERIFY` 分类异常。
  - 任务与异常能形成定位闭环。
- 通过依据：
  - `services/center/internal/jobs` VERIFY 作业集成测试通过

## 6. 阶段6：后台清理

### 6.1 删除文件进入后台清理作业而非同步直接删除

- 状态：已通过
- 前置条件：
  - 文件已扫描入库
  - 中心服务持续运行
- 操作步骤：
  1. 打开客户端，进入“文件中心”。
  2. 选中一个本地文件资产，执行删除。
  3. 观察文件中心提示、任务中心和最终文件状态。
- 预期结果：
  - 删除动作会进入后台清理，而不是同步即时删库删文件。
  - 任务中心出现 `DELETE_ASSET` 清理任务。
  - 任务完成后，物理文件被删除。
- 通过依据：
  - `services/center/internal/assets` 后台清理集成测试通过
