# 统一文件管理系统-网盘同步任务真实客户端测试用例清单

## 文档说明

- 更新时间：2026-04-13
- 适用范围：`client`、`services/center`
- 关联计划文档：[统一文件管理系统-网盘同步任务状态规划与任务拆分方案.md](/B:/new_project/mare/docs/实现路径/统一文件管理系统-网盘同步任务状态规划与任务拆分方案.md)
- 目标：为后续 AI 实现和人工联调提供一份可直接执行的“真实客户端测试用例清单”

## 1. 测试目标

本轮测试不是泛泛验证网盘功能，而是围绕本次计划中准备修改的内容，确认以下目标成立：

- `CloudDrive2` 上传不会在预处理或排队阶段错误显示传输进度
- `aria2` 下载不会因为错误兜底导致进度假满到 `100%`
- 任务中心中的状态、阶段、进度、速度、ETA 与真实链路一致
- 文件中心中的端点状态投影与任务中心结果一致
- 暂停、继续、取消、重试、恢复逻辑不会破坏上述规则
- 崩溃或重启后，上传/下载能够按预期恢复
- 真实 `20MB` 文件能够完成上传和下载
- 本轮修改不会回归影响已有上传、下载和删副本的展示

## 2. 测试范围

### 2.1 本轮范围内

- 文件中心：
  - 上传到 `115`
  - 从 `115` 下载
  - 删除 `115` 副本后的状态投影
- 任务中心：
  - 一级任务与二级任务显示
  - 阶段、进度、速度、ETA 展示
  - 暂停、继续、取消、重试
  - 恢复逻辑
- 后端链路：
  - `CD2_REMOTE_UPLOAD`
  - `ARIA2`
  - `Job / JobItem / ExternalTask / EndpointProjection`

### 2.2 本轮明确不覆盖

- 导入中心 `CLOUD` 目标端正式开放
- 异常中心、通知中心的专项改造
- 与本次进度异常无直接关系的普通 `COPY` 链路重构
- 与本轮无关的设置页、存储节点页专项交互优化

## 3. 测试环境与联调前提

### 3.1 基础环境

- 客户端可正常打开文件中心与任务中心
- 中心服务已启动
- `CloudDrive2` 网关配置有效
- `aria2` 运行组件在线
- 已存在可用的 `115` 云节点
- 文件中心中存在可用于上传与下载的测试素材

### 3.2 真实联调前提

- 必须准备一个可写入的 `115` 目标目录
- 必须准备一个可读写的本地或 NAS 目标目录
- 必须准备至少两个真实测试文件：
  - 小文件：便于快速验证状态切换
  - `20MB` 文件：用于真实传输完成验证
- 必须能够触发服务重启或进程重启，用于恢复场景验证

### 3.3 验证方式说明

本清单按 4 类测试层级标注：

- `前端集成`：客户端页面与状态投影逻辑
- `后端集成`：中心服务、任务域、集成层联动
- `E2E`：从文件中心发起到任务中心观察的完整用户链路
- `人工验收`：依赖真实 `115 / CD2 / aria2` 环境的联调

## 5. 文件中心场景

### 5.1 从文件中心发起上传到 115

- 优先级：`P0`
- 测试层级：`前端集成`、`E2E`、`人工验收`
- 目标：
  - 用户能从文件中心明确发起“上传到 115”
  - 文件中心端点状态在任务创建后进入 `同步中`
- 关键检查点：
  - 动作入口文案正确
  - 动作禁用逻辑正确
  - 发起后任务中心出现对应任务
  - 文件中心端点状态变为 `同步中`

### 5.2 从文件中心发起从 115 下载

- 优先级：`P0`
- 测试层级：`前端集成`、`E2E`、`人工验收`
- 目标：
  - 用户能明确发起“从 115 下载”
  - 文件中心端点状态与任务中心一致
- 关键检查点：
  - 动作文案不与普通复制混淆
  - 发起后任务中心出现 `DOWNLOAD` 任务
  - 文件中心端点状态正确进入 `同步中`

### 5.3 文件与目录的端点状态投影差异

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`
- 目标：
  - 文件不出现 `部分同步`
  - 目录允许出现 `部分同步`
- 关键检查点：
  - 单文件条目只有 `未同步 / 同步中 / 已同步`
  - 目录条目在部分完成时显示 `部分同步`

### 5.4 删除副本后的端点状态回退

- 优先级：`P1`
- 测试层级：`前端集成`、`后端集成`、`人工验收`
- 目标：
  - 删除 `115` 副本后，文件中心端点状态正确回落
- 关键检查点：
  - 删除副本创建真实任务
  - 任务中心显示正确
  - 文件中心端点从 `已同步` 回落到 `未同步` 或目录级 `部分同步`

## 6. CloudDrive2 上传链路场景

### 6.1 预处理阶段不展示错误进度

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`
- 目标：
  - `WaitforPreprocessing / Preprocessing / Inqueue` 阶段不推进百分比
- 关键检查点：
  - 二级任务显示阶段文案
  - 不显示误导性的进度推进
  - 不显示伪精确速度

### 6.2 进入 Transfer 后才开始展示有效进度

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`、`人工验收`
- 目标：
  - 只有进入 `Transfer` 后，进度条才开始真实推进
- 关键检查点：
  - `Transfer` 前进度冻结或不生效
  - `Transfer` 后 `bytesDone / size` 推进
  - 一级任务总进度随之变化

### 6.3 上传暂停与继续

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`、`E2E`、`人工验收`
- 目标：
  - 暂停后内部状态、外部状态和页面显示一致
  - 继续后恢复到正确阶段
- 关键检查点：
  - 暂停后显示 `已暂停`
  - 暂停后速度消失、进度冻结
  - 继续后恢复 `QUEUED / RUNNING`
  - 恢复后重新进入正确阶段

### 6.4 上传取消与失败投影

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`
- 目标：
  - `Cancelled / Skipped / Error / FatalError` 正确映射到 UI
- 关键检查点：
  - 取消后不再推进进度
  - `Skipped` 不被显示成成功
  - 失败时“失败位置 / 外部状态 / 原始错误”正确

### 6.5 多上传任务并发下的速度展示

- 优先级：`P1`
- 测试层级：`前端集成`、`人工验收`
- 目标：
  - 一级任务可展示总上传速度
  - 二级任务不显示伪精确单任务速度
- 关键检查点：
  - 并发上传时一级任务速度不为空
  - 二级任务速度字段不误导

## 7. aria2 下载链路场景

### 7.1 waiting 阶段不展示有效进度和 ETA

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`
- 目标：
  - `waiting` 仅表示等待下载，不显示误导性进度和 ETA
- 关键检查点：
  - 详情页无有效 ETA
  - 不出现错误的快速进度推进

### 7.2 active 阶段展示真实进度、速度、ETA

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`、`人工验收`
- 目标：
  - `active` 阶段字段与真实下载一致
- 关键检查点：
  - 进度按真实下载推进
  - 速度显示真实值或合理推导值
  - ETA 随剩余字节变化

### 7.3 下载假满修复

- 优先级：`P0`
- 测试层级：`后端集成`、`前端集成`、`E2E`
- 目标：
  - 下载过程中不再因目标文件大小而直接接近 `100%`
- 关键检查点：
  - `bytesDone` 不再无条件使用目标文件大小兜底
  - 详情刷新后进度仍保守且正确
  - 一级任务聚合进度不虚高

### 7.4 downloadSpeed 为 0 时的速度推导

- 优先级：`P0`
- 测试层级：`后端集成`
- 目标：
  - 当 `downloadSpeed=0` 但 `completedLength` 持续增长时，能合理推导速度
- 关键检查点：
  - 推导速度为正
  - 不覆盖已有更可靠速度
  - 不影响 ETA 计算边界

### 7.5 下载暂停、继续、取消、失败

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`、`E2E`、`人工验收`
- 目标：
  - `paused / removed / error / complete` 四类状态投影准确
- 关键检查点：
  - 暂停后显示 `已暂停`
  - 继续后回到 `waiting / active`
  - 取消后显示 `已取消`
  - 失败后显示 `下载失败`

## 8. 任务中心显示场景

### 8.1 一级任务与二级任务的角色分离

- 优先级：`P0`
- 测试层级：`前端集成`
- 目标：
  - 一级任务只展示内部调度状态
  - 二级任务展示“内部状态 + 外部阶段”
- 关键检查点：
  - 一级任务不直接把外部状态当主状态
  - 二级任务阶段文案可解释

### 8.2 一级任务总进度与二级任务聚合一致

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`
- 目标：
  - 一级任务总进度与二级任务有效进度聚合一致
- 关键检查点：
  - 不可信阶段不参与聚合
  - 完成态后聚合收敛到 `100%`

### 8.3 速度与 ETA 的展示门控

- 优先级：`P0`
- 测试层级：`前端集成`
- 目标：
  - 只在可信阶段显示速度和 ETA
- 关键检查点：
  - 上传前置阶段不显示 ETA
  - 下载 `waiting / paused` 阶段不显示 ETA
  - 完成态与取消态不显示运行速度

### 8.4 详情抽屉、列表行、展开子项一致性

- 优先级：`P0`
- 测试层级：`前端集成`
- 目标：
  - 同一任务在 3 个视图中的状态解释一致
- 关键检查点：
  - 列表行、详情抽屉、展开子项的文案一致
  - 错误信息、外部状态、失败位置不互相冲突

## 9. 任务控制场景

### 9.1 主任务暂停、继续、取消、重试

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`、`E2E`
- 目标：
  - 一级任务控制后，内部状态和子项状态按规则变化
- 关键检查点：
  - 按钮显隐正确
  - 状态变化正确
  - 重试后非完成项进入 `WAITING_RETRY -> QUEUED`

### 9.2 子任务暂停、继续、取消

- 优先级：`P0`
- 测试层级：`前端集成`、`后端集成`
- 目标：
  - 二级任务控制行为正确，且不破坏一级任务聚合
- 关键检查点：
  - 子项暂停后一级任务进入正确调度态
  - 子项继续后恢复排队
  - 子项取消后不再推进

## 10. 恢复与重启场景

### 10.1 上传进行中后服务重启，恢复后继续上传

- 优先级：`P0`
- 测试层级：`后端集成`、`人工验收`
- 目标：
  - 服务重启后上传任务能恢复并继续完成
- 关键检查点：
  - 内部状态 `RUNNING -> QUEUED`
  - 能附着旧 `CD2` 任务或按规则恢复
  - 最终仍可成功上传完成

### 10.2 下载进行中后服务重启，恢复后继续下载

- 优先级：`P0`
- 测试层级：`后端集成`、`人工验收`
- 目标：
  - 服务重启后下载任务能恢复并继续完成
- 关键检查点：
  - 能复用旧 `aria2` 任务或重新发现目标任务
  - 最终仍可成功下载完成

### 10.3 暂停态重启后不会自动继续

- 优先级：`P1`
- 测试层级：`后端集成`
- 目标：
  - `PAUSED` 状态重启后仍保持等待继续

### 10.4 终态任务不会被错误恢复

- 优先级：`P1`
- 测试层级：`后端集成`
- 目标：
  - `COMPLETED / FAILED / CANCELED` 重启后不会被错误恢复
