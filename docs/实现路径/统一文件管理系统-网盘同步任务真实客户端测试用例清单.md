# 统一文件管理系统-网盘同步任务真实客户端测试用例清单
## 文档说明

- 更新时间：2026-04-13
- 适用范围：`client`、`services/center`
- 关联计划文档：[统一文件管理系统-网盘同步任务状态规划与任务拆分方案.md](/B:/new_project/mare/docs/实现路径/统一文件管理系统-网盘同步任务状态规划与任务拆分方案.md)
- 目标：为后续 AI 实现和人工联调提供一份可直接执行的真实客户端测试清单，不只覆盖进度显示，还覆盖恢复与韧性问题。

## 1. 测试目标

本轮测试的目标是确认以下结论同时成立：

- `CloudDrive2` 上传不会在预处理或排队阶段错误显示传输进度
- `aria2` 下载不会再因为错误兜底导致进度假满到 `100%`
- `CD2` 上传通道断线不会立刻把本地会话全部判死
- `aria2` 恢复时不会继续复用错误态旧任务
- 传输进度高频刷新下，系统不会因为每秒重事务而明显卡顿
- 任务中心中的状态、阶段、进度、速度、ETA 与真实链路一致
- 文件中心中的端点状态投影与任务中心结果一致
- 暂停、继续、取消、重试、重启恢复不会破坏上述规则
- 真实 `20MB` 文件能够完成上传和下载

## 2. 测试范围

### 2.1 本轮覆盖

- 文件中心：
  - 上传到 `115`
  - 从 `115` 下载
  - 删除 `115` 副本后的状态投影
- 任务中心：
  - 一级任务与二级任务显示
  - 阶段、进度、速度、ETA 展示
  - 暂停、继续、取消、重试
  - 重启恢复
- 后端链路：
  - `CD2_REMOTE_UPLOAD`
  - `ARIA2`
  - `Job / JobItem / ExternalTask / EndpointProjection`

### 2.2 本轮不覆盖

- CD2 官方离线下载业务编排
- `CLOUD -> CLOUD` 正式同步
- 与本轮问题无直接关系的普通 `COPY` 链路重构
- 导入中心、异常中心、通知中心的专项改造

## 3. 环境前提

### 3.1 基础环境

- 客户端可正常打开文件中心与任务中心
- 中心服务已启动
- `CloudDrive2` 网关配置有效
- `aria2` 运行组件在线
- 已存在可用的 `115` 云节点
- 文件中心中存在可用于上传与下载的测试素材

### 3.2 真实联调前提

- 准备一个可写入的 `115` 目标目录
- 准备一个可读写的本地或 NAS 目录
- 至少准备两个真实测试文件：
  - 小文件：便于快速验证阶段切换
  - `20MB` 文件：用于真实传输完成验证
- 能够主动触发中心服务重启，用于恢复场景验证

## 4. 优先级定义

- `P0`：本轮必须覆盖，缺失则无法证明修复有效
- `P1`：重要场景，建议本轮完成
- `P2`：增强与边角场景，可在本轮末尾补齐

## 5. 自动化验证项

### 5.1 后端自动化

- `P0` `aria2` 恢复状态边界
  - 覆盖 `active / waiting / paused / complete / error / removed`
  - 期望 `error / removed` 不再被复用
- `P0` 传输进度节流
  - 高频回调下不再每次都落库
  - 首次上报、超过 5 秒、超过 5% 进度增量时才持久化
- `P0` `CD2` 通道断线容错
  - `RemoteUploadChannel` 断线后不立即关闭本地上传会话
  - 等待侧仍允许后续重建通道
- `P1` 云副本不能作为上传源
  - `CLOUD -> CLOUD` 在当前实现下不得被视作已支持上传链路

### 5.2 前端自动化

- `P0` 上传前置阶段展示
  - `WaitforPreprocessing / Preprocessing / Inqueue` 只显示阶段，不显示误导性的进度推进
- `P0` 下载失败阶段展示
  - `aria2 error` 显示失败位置、外部状态和原始错误
- `P1` 详情、列表、展开子项口径一致
  - 同一任务在 3 处视图中的阶段解释一致

## 6. 真实客户端人工验证项

### 6.1 从文件中心发起上传到 115

- 优先级：`P0`
- 步骤：
  1. 在文件中心选择本地或 NAS 文件
  2. 发起同步到 `115`
  3. 跳转到任务中心查看任务
- 期望：
  - 文件中心目标端点进入 `同步中`
  - 任务中心出现 `UPLOAD` 路由任务
  - `Transfer` 前不出现误导性的进度推进

### 6.2 从文件中心发起从 115 下载

- 优先级：`P0`
- 步骤：
  1. 在文件中心选择 `115` 副本
  2. 发起同步到本地或 NAS
  3. 跳转到任务中心查看任务
- 期望：
  - 文件中心目标端点进入 `同步中`
  - 任务中心出现 `DOWNLOAD` 路由任务
  - `waiting` 阶段不显示伪进度和 ETA
  - `active` 后进度、速度、ETA 才开始可信变化

### 6.3 20MB 文件上传验证

- 优先级：`P0`
- 步骤：
  1. 用真实 `20MB` 文件发起上传到 `115`
  2. 观察任务中心阶段切换
  3. 等待完成后检查 `115` 目标目录
- 期望：
  - 阶段经历“预处理/排队 -> 上传中 -> 已完成”
  - 只有 `Transfer` 阶段才显示有效进度
  - 完成后文件在 `115` 真实存在且大小正确
  - 文件中心端点最终显示 `已同步`

### 6.4 20MB 文件下载验证

- 优先级：`P0`
- 步骤：
  1. 用真实 `20MB` 云端文件发起下载到本地
  2. 观察任务中心
  3. 完成后检查本地目标文件
- 期望：
  - 不出现“未下完但 100%”
  - `active` 阶段速度和 ETA 变化合理
  - 完成后本地文件存在且大小正确
  - 文件中心端点最终显示 `已同步`

## 7. 控制与恢复场景

### 7.1 上传暂停与继续

- 优先级：`P0`
- 步骤：
  1. 在上传进行中点击暂停
  2. 确认状态冻结
  3. 再点击继续
- 期望：
  - 暂停后任务显示 `已暂停`
  - 速度消失、进度冻结
  - 继续后恢复到正确阶段

### 7.2 下载暂停与继续

- 优先级：`P0`
- 期望：
  - 暂停后 `aria2` 外部状态与内部状态一致
  - 继续后恢复到 `waiting / active`

### 7.3 上传进行中服务重启

- 优先级：`P0`
- 步骤：
  1. 在上传进行中重启中心服务
  2. 服务恢复后观察任务
- 期望：
  - 内部任务先回到 `QUEUED`
  - 本地上传会话不会因为通道中断被立即整体判死
  - 后续允许继续等待并恢复完成

### 7.4 下载进行中服务重启

- 优先级：`P0`
- 步骤：
  1. 在下载进行中重启中心服务
  2. 观察恢复后的任务
- 期望：
  - 不再直接复用错误态旧任务
  - 恢复后继续有效下载或重新入队

## 8. 删除副本与回归场景

### 8.1 删除副本不误套用上传/下载指标逻辑

- 优先级：`P1`
- 期望：
  - 删除副本任务不会错误显示上传/下载进度
  - 文件中心端点状态回落正确

### 8.2 普通 COPY 链路不受本轮修改影响

- 优先级：`P0`
- 期望：
  - 本地/NAS 之间复制的既有行为不回归

### 8.3 高频刷新下任务中心仍可正常响应

- 优先级：`P1`
- 期望：
  - 传输任务运行中，任务中心不会因为进度回写过于频繁而明显卡顿
  - 列表刷新与详情打开响应保持可接受

## 10. 通过标准

当以下条件同时成立时，本轮测试视为通过：

- 上传前置阶段不再错误显示传输进度
- 下载过程中不再出现假满到 `100%`
- `CD2` 通道断线不会立刻本地全量失败
- `aria2` 恢复不再复用错误态旧任务
- 进度高频刷新下系统没有明显卡顿或异常阻塞
- 任务中心中的状态、阶段、进度、速度、ETA 解释一致
- 文件中心端点状态与任务中心结果一致
- 真实 `20MB` 文件上传与下载都能完成
- 删除副本状态投影与普通 `COPY` 链路均未回归
