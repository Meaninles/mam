# 统一文件管理系统-切片11页面收口真实客户端测试用例

## 文档说明

- 目的：记录切片 11 页面收口过程中的真实客户端手工测试用例。
- 约束：只记录本轮“页面收口”新增或回补的验证项，不和既有的网盘同步测试清单混写。
- 使用方式：每完成一个页面，就在对应章节追加真实操作步骤、预期结果和失败定位提示。

## 1. 设置页

### 1.1 依赖服务状态展示

- 前置条件：
  - 中心服务可访问。
  - `CloudDrive2` 运行态为 `ONLINE`。
  - `aria2` 运行态为 `ERROR`，且运行态接口返回 `lastErrorCode` 与 `lastErrorMessage`。
- 操作步骤：
  1. 打开客户端，进入“设置”。
  2. 切换到“依赖服务”子页。
  3. 观察 `CloudDrive2` 卡片、`aria2` 卡片和右侧汇总卡片。
- 预期结果：
  - 页面同时展示 `CloudDrive2` 和 `aria2` 两个组件。
  - `CloudDrive2` 状态显示“在线”。
  - `aria2` 状态显示“异常”。
  - `CloudDrive2` 显示“凭据状态”。
  - `aria2` 显示“最近错误代码”和“最近错误”。
  - 汇总卡片同步显示两者的运行状态。
- 失败定位：
  - 若只显示 `CloudDrive2`，优先检查 `/api/integrations/runtime` 是否返回 `aria2` 组件。
  - 若不显示错误代码，优先检查前端是否消费 `lastErrorCode`。

### 1.2 CloudDrive2 保存后状态回写

- 前置条件：
  - 中心服务可访问 `CloudDrive2` 网关保存和测试接口。
- 操作步骤：
  1. 打开“设置 -> 依赖服务”。
  2. 输入 `CloudDrive2` 账号和密码。
  3. 点击“保存 CloudDrive2”。
- 预期结果：
  - 页面出现“CloudDrive2 配置已保存，并已确认连接正常”之类的成功反馈。
  - `CloudDrive2` 指示灯切换为成功态。
  - “凭据状态”切换为“已保存凭据”。
- 失败定位：
  - 若保存成功但状态未刷新，优先检查保存后是否补跑连接测试并回写本地运行态。

## 2. 存储节点页

### 2.1 115 节点治理信息展示

- 前置条件：
  - 已存在一个 115 网盘节点。
  - 节点具备 `Token 状态 / 账号别名 / 最近鉴权时间 / 最近错误代码 / 最近错误信息`。
- 操作步骤：
  1. 打开客户端，进入“存储节点”。
  2. 切换到“网盘管理”。
  3. 在 115 节点列表中找到目标节点。
- 预期结果：
  - 列表除“鉴权状态”外，还展示 `Token 状态`。
  - 列表展示 `账号别名`。
  - 列表展示 `最近鉴权时间` 与 `最近鉴权结果`。
  - 若存在失败，列表直接展示最近失败原因。
- 失败定位：
  - 若只显示状态和最近测试时间，优先检查云节点列表接口是否返回治理字段。
  - 若接口已返回但页面没显示，优先检查 `CloudTable` 渲染逻辑。

### 2.2 编辑网盘时的凭据反馈

- 前置条件：
  - 已存在一个 115 网盘节点。
  - 节点当前有 `Token 状态` 或最近失败信息。
- 操作步骤：
  1. 打开“存储节点 -> 网盘管理”。
  2. 点击目标 115 节点的“编辑”。
  3. 观察编辑抽屉顶部表单区域。
- 预期结果：
  - 抽屉显示“当前凭据状态”摘要块。
  - 摘要块展示 `Token 状态`。
  - 若存在 `账号别名 / 最近鉴权时间 / 最近鉴权结果 / 最近错误代码 / 最近失败原因`，都能看到。
  - 用户无需离开编辑态即可判断是否要重新扫码或重填 Token。
- 失败定位：
  - 若编辑态没有凭据摘要，优先检查 `cloudRecordToDraft` 是否把治理字段带入草稿。
  - 若草稿有值但 UI 未显示，优先检查编辑抽屉摘要块渲染条件。

### 2.3 已保存扫码凭据切换到 Token 时预填

- 前置条件：
  - 目标 115 节点最初通过“扫码登录获取 Token”方式保存过，后台已保存刷新令牌或 Cookie。
- 操作步骤：
  1. 进入“存储节点 -> 网盘管理”。
  2. 编辑目标节点。
  3. 将接入方式从“扫码登录获取 Token”切换到“填入 Token”。
- 预期结果：
  - `网盘 Token` 输入框自动带出已保存的凭据。
  - 用户可以在原有凭据基础上决定是否替换。
- 失败定位：
  - 若切换后输入框为空，优先检查列表接口是否返回 `token`，以及 `cloudRecordToDraft` 是否保留该值。

## 3. 文件中心页

### 3.1 云端动作前置校验

- 前置条件：
  - `CloudDrive2` 或 `aria2` 至少有一个运行态异常，或 115 节点未完成鉴权。
- 操作步骤：
  1. 打开客户端，进入“文件中心”。
  2. 找到带有 115 端点的文件或文件夹。
  3. 观察页面顶部提示卡和 115 端点按钮状态。
- 预期结果：
  - 页面出现“云端动作受限”提示卡。
  - 提示卡明确说明是 `CloudDrive2`、`aria2` 还是 115 鉴权导致不可执行。
  - 提示卡提供“打开设置页”或“打开存储节点”的跳转入口。
  - 115 端点的同步或删除副本动作被禁用，不会误导用户继续点击。
- 失败定位：
  - 若云端动作仍可点击，优先检查文件中心是否消费了统一的 `integrationHealth`。
  - 若提示卡没有跳转入口，优先检查动作聚合逻辑是否返回 `actions`。

### 3.2 云端动作语义确认

- 前置条件：
  - `CloudDrive2`、`aria2` 与 115 节点鉴权均正常。
- 操作步骤：
  1. 在文件中心打开一个存在 115 副本的文件。
  2. 分别观察“同步”“删除”菜单下对 115 的动作。
- 预期结果：
  - 同步到 115 时，页面语义明确是“上传到 115”。
  - 从 115 下行到本地或 NAS 时，页面语义明确是“从 115 下载”。
  - 删除 115 时，页面语义明确是“删除 115 副本”。
- 失败定位：
  - 若动作仍然显示成通用“复制”，优先检查文件中心动作计划摘要。

## 4. 任务中心页

### 4.1 CloudDrive2 外部执行器可观测性

- 前置条件：
  - 存在一条 `CD2_REMOTE_UPLOAD` 传输任务。
- 操作步骤：
  1. 打开客户端，进入“任务中心 -> 传输任务”。
  2. 找到一条上传到 115 的任务并打开详情。
- 预期结果：
  - 列表中的当前阶段显示为 CloudDrive2 语义阶段，而不是原始代码。
  - 详情展示 `执行引擎`、`外部任务 ID`、`外部状态`。
  - 当任务处于缓存或排队阶段时，不会被误显示为“已完成”。
- 失败定位：
  - 若只看到内部状态，看不到外部执行器字段，优先检查任务详情 DTO 映射。

### 4.2 aria2 失败定位

- 前置条件：
  - 存在一条 `ARIA2` 下载失败任务。
- 操作步骤：
  1. 打开“任务中心 -> 传输任务”。
  2. 找到一条从 115 下载失败的任务并打开详情。
- 预期结果：
  - 列表当前阶段显示为“aria2 下载失败”。
  - 详情展示 `执行引擎`、`外部任务 ID`、`外部状态`、`失败位置`、`原始错误`。
  - 运维可以直接判断失败发生在 `aria2`，而不是中心服务内部。
- 失败定位：
  - 若仍需查看服务端日志才能判断位置，优先检查 `failureLocation` 和 `rawErrorMessage` 是否已投影到页面。

## 5. 异常中心页

### 5.1 云端问题分类展示

- 前置条件：
  - 至少存在一条 `CloudDrive2`、`aria2` 或 115 相关异常。
- 操作步骤：
  1. 打开客户端，进入“异常中心”。
  2. 打开一条云端相关异常详情。
- 预期结果：
  - 详情中出现“云端问题类型”字段。
  - `CloudDrive2` 类问题显示为“CloudDrive2 网关问题”。
  - `aria2` 类问题显示为“aria2 下载器问题”。
  - 115 凭据类问题显示为“115 鉴权问题”。
- 失败定位：
  - 若没有云端问题类型字段，优先检查异常中心是否调用统一的云端问题分类 helper。

### 5.2 直达设置页或存储节点

- 前置条件：
  - 存在一条 `CloudDrive2` 或 `aria2` 类异常，以及一条 115 鉴权类异常。
- 操作步骤：
  1. 分别打开这两类异常的详情。
  2. 观察底部治理入口按钮。
- 预期结果：
  - `CloudDrive2` / `aria2` 类异常显示“打开设置页”。
  - 115 鉴权类异常保留“打开存储节点”。
- 失败定位：
  - 若 `CloudDrive2` / `aria2` 仍只跳存储节点，优先检查异常详情按钮判定逻辑。

## 6. 通知中心页

### 6.1 CloudDrive2 / aria2 运行态通知跳转

- 前置条件：
  - 至少存在一条 `CloudDrive2` 或 `aria2` 运行态问题通知。
- 操作步骤：
  1. 打开客户端通知中心。
  2. 点击该通知的主按钮“去处理”。
  3. 再打开更多操作菜单。
- 预期结果：
  - 主按钮直接把用户带到“设置 -> 依赖服务”。
  - 更多操作菜单中出现“打开设置页”。
- 失败定位：
  - 若仍跳异常中心，优先检查通知中心是否按云端问题类型重定向默认目标。

### 6.2 115 鉴权失效通知跳转

- 前置条件：
  - 至少存在一条 115 凭据失效通知。
- 操作步骤：
  1. 打开通知中心。
  2. 点击该通知的主按钮“去处理”。
- 预期结果：
  - 用户被带到“存储节点 -> 网盘管理”。
  - 不会误跳到设置页。
- 失败定位：
  - 若 115 凭据问题被错误带到设置页，优先检查通知中心云端问题类型与目标映射。

## 7. 导入中心页

### 7.1 CLOUD 目标端可见性

- 前置条件：
  - 资产库下至少存在一个 `CLOUD` 挂载目标。
  - 该挂载目标已完成基本可用性校验。
- 操作步骤：
  1. 打开客户端，进入“导入中心”。
  2. 选择一个可导入设备会话。
  3. 观察当前层级文件列表的目标端分配区域。
- 预期结果：
  - 目标端列表中出现 `115 云归档` 之类的 `CLOUD` 目标。
  - 目标端类型语义显示为 `115网盘`。
  - 用户可以像本地/NAS 一样勾选该目标端。
- 失败定位：
  - 若看不到 `CLOUD` 目标，优先检查导入服务 `loadTargetEndpoints` 是否已包含 `CLOUD` 挂载。

### 7.2 把导入对象分配到 115 网盘

- 前置条件：
  - 导入中心已显示 `115网盘` 目标端。
- 操作步骤：
  1. 在导入中心打开一个设备会话。
  2. 在当前层级选择一个文件。
  3. 勾选 `115 云归档` 目标端。
  4. 继续点击“提交导入”并确认。
- 预期结果：
  - 勾选动作会立即写回该对象的目标端选择。
  - 提交确认弹窗可以正常出现。
  - 后续任务创建时，目标端包含 `CLOUD` 挂载。
- 失败定位：
  - 若勾选无效，优先检查前端 `onSaveSelectionTargets` 是否带上 `target-cloud`。
  - 若提交后未进入任务，优先检查服务端 `loadTargetsByIDs` 是否允许 `CLOUD` 目标参与编排。
